### UE_06.0_1: Referenzen vs. Primitive

Analysiere folgenden Code:

```java
public class ReferenzTest {
    public static void main(String[] args) {
        int a = 10;
        int b = a;
        b = 20;
        System.out.println("a = " + a + ", b = " + b);

        Patient p1 = new Patient("Anna", 25);
        Patient p2 = p1;
        p2.setName("Ben");
        System.out.println("p1: " + p1.getName() + ", p2: " + p2.getName());
    }
}
```

1. Was wird ausgegeben? Begründe die unterschiedlichen Ergebnisse.
2. Erstelle ein Diagramm, das die Speichersituation zeigt (primitiv vs. Referenz)

### UE_06.0_2: Methodenparameter

```java
public class ParameterTest {
    public static void aendereAlter(Patient p) {
        p.setAlter(50);
    }

    public static void aendereZahl(int x) {
        x = 100;
    }

    public static void main(String[] args) {
        Patient p = new Patient("Anna", 25);
        aendereAlter(p);
        System.out.println(p.getAlter());  // ?

        int zahl = 25;
        aendereZahl(zahl);
        System.out.println(zahl);          // ?
    }
}
```

1. Was wird ausgegeben? Begründe.
2. Was bedeutet "Call by Value" in Java? Warum verhält sich der Patient anders als der int?

### UE_06.0_4: Immutable Befund-Klasse

Erstelle eine immutable Klasse `Befund`:

- `final`-Klasse
- Alle Attribute sind `private final`:
  - `patientId` (String)
  - `diagnose` (String)
  - `messwert` (double)
  - `datum` (String)
- Einziger Konstruktor (setzt alle Werte)
- Nur Getter (keine Setter)
- Eine Methode `ausgabe()`, die alle Werte ausgibt

Erstelle in der `main`-Methode einen Befund und weise ihn einer zweiten Variable zu. Zeige, dass eine Änderung nicht möglich ist.
