# UE_11.1 JPA Grundlagen – Übungen

[<<](../skriptum/11.1_JPA_Grundlagen.md)

### UE_11.1_1: Projekt und persistence.xml einrichten

Erstelle ein neues Java-Projekt mit JPA/Hibernate:

1. Füge die benötigten Abhängigkeiten hinzu (Hibernate, MySQL Connector)
2. Erstelle `src/main/resources/META-INF/persistence.xml` mit einer `persistence-unit` namens `medizinPU`
3. Konfiguriere die Verbindung zu einer MySQL-Datenbank `medizin`
4. Setze `hibernate.hbm2ddl.auto` auf `update`

### UE_11.1_2: Entity-Klasse erstellen

Erstelle eine Entity-Klasse `Patient` im Package `at.htl.medizin.model` mit:

- `id` (int, `@Id`, `@GeneratedValue(strategy = IDENTITY)`)
- `name` (String, `@Column(nullable = false)`)
- `alter` (int)
- `diagnose` (String)
- Standard-Konstruktor, parametrisierten Konstruktor, Getter/Setter
- `toString()`

### UE_11.1_3: Patienten speichern (persist)

Schreibe ein Programm `PatientPersistieren.java`:

1. Erzeuge einen `EntityManager` über die persistence-unit `medizinPU`
2. Erstelle 3 `Patient`-Objekte
3. Speichere sie mit `em.persist()` in einer Transaktion
4. Gib nach dem Commit die generierten IDs aus
5. Schließe `EntityManager` und `EntityManagerFactory`

### UE_11.1_4: Patienten abrufen (find)

Schreibe ein Programm `PatientFinden.java`:

1. Frage den Benutzer nach einer ID (mit `Scanner`)
2. Rufe den Patienten mit `em.find(Patient.class, id)` ab
3. Gib die Patientendaten aus oder eine Meldung, falls nicht gefunden

### UE_11.1_5: JPQL-Abfragen

Schreibe ein Programm `PatientJpql.java` mit folgenden Abfragen:

1. Alle Patienten: `SELECT p FROM Patient p`
2. Patienten mit einer bestimmten Diagnose: `SELECT p FROM Patient p WHERE p.diagnose = :diag`
3. Patienten über einem bestimmten Alter: `SELECT p FROM Patient p WHERE p.alter > :min`
4. Anzahl der Patienten: `SELECT COUNT(p) FROM Patient p`

### UE_11.1_6: Patienten aktualisieren (merge)

Schreibe ein Programm `PatientAktualisieren.java`:

1. Rufe einen Patienten mit `em.find()` ab (managed)
2. Ändere seine Diagnose
3. Führe `commit()` aus – die Änderung sollte automatisch gespeichert werden
4. Rufe den Patienten erneut ab und gib die aktualisierten Daten aus

### UE_11.1_7: Patienten löschen (remove)

Schreibe ein Programm `PatientLoeschen.java`:

1. Frage den Benutzer nach einer ID
2. Lösche den Patienten mit `em.remove()` in einer Transaktion
3. Gib aus, ob der Patient gelöscht wurde oder nicht existiert

### UE_11.1_8: Transaktionen verstehen

Schreibe ein Programm `TransaktionsTest.java`:

1. Beginne eine Transaktion
2. Speichere einen neuen Patienten
3. Löse bewusst einen Fehler aus (z. B. `int x = 1 / 0`)
4. Fange den Fehler und führe `rollback()` aus
5. Überprüfe, dass der Patient nicht in der Datenbank ist
6. Führe denselben Code ohne Fehler aus – der Patient sollte gespeichert werden
