### UE_11.2_1: Datenbanktabellen anlegen

Erstelle mit einem MySQL-Client folgende Tabellen:

```sql
CREATE TABLE patient (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL
);

CREATE TABLE adresse (
    id INT AUTO_INCREMENT PRIMARY KEY,
    strasse VARCHAR(100),
    stadt VARCHAR(100),
    patient_id INT UNIQUE,
    FOREIGN KEY (patient_id) REFERENCES patient(id)
);

CREATE TABLE befund (
    id INT AUTO_INCREMENT PRIMARY KEY,
    diagnose VARCHAR(200),
    patient_id INT,
    FOREIGN KEY (patient_id) REFERENCES patient(id)
);
```

### UE_11.2_2: Entity-Klassen mit @ManyToOne

Erstelle eine Entity `Befund` mit:
- `id` (int, `@Id`, `@GeneratedValue`)
- `diagnose` (String)
- `patient` (`@ManyToOne(fetch = FetchType.LAZY)`, `@JoinColumn(name = "patient_id")`)

Ergänze `Patient` um:
- `befunde` (`@OneToMany(mappedBy = "patient", cascade = CascadeType.ALL, orphanRemoval = true)`)

### UE_11.2_3: Patienten mit Befunden speichern

Schreibe ein Programm `PatientMitBefund.java`:

1. Erstelle einen Patienten mit 2–3 Befunden
2. Setze die Beziehung auf beiden Seiten (`b.setPatient(p)` und `p.getBefunde().add(b)`)
3. Speichere den Patienten mit `em.persist(p)` – die Befunde sollten durch Cascade mitgespeichert werden
4. Gib die generierten IDs aus

### UE_11.2_4: Befunde lazy laden

Schreibe ein Programm `LazyLoadTest.java`:

1. Rufe einen Patienten mit `em.find()` ab
2. Greife innerhalb der Transaktion auf `p.getBefunde()` zu (z. B. `p.getBefunde().size()`)
3. Greife nach dem `commit()` auf `p.getBefunde()` zu – beobachte die `LazyInitializationException`
4. Kommentiere den Zugriff nach dem commit aus

### UE_11.2_5: JOIN FETCH in JPQL

Schreibe ein Programm `JoinFetchTest.java`:

1. Führe eine JPQL-Abfrage mit `JOIN FETCH` aus:
   ```java
   TypedQuery<Patient> query = em.createQuery(
       "SELECT p FROM Patient p JOIN FETCH p.befunde WHERE p.id = :id", Patient.class);
   ```
2. Greife nach dem `commit()` auf `p.getBefunde()` zu – es sollte funktionieren
3. Gib alle Befunde des Patienten aus

### UE_11.2_6: @OneToOne – Adresse

Erstelle eine Entity `Adresse` mit `@OneToOne` zu `Patient`:

1. `Adresse` hat `id`, `strasse`, `stadt` und `patient` (`@OneToOne`, `@JoinColumn(name = "patient_id")`)
2. Ergänze `Patient` um `adresse` (`@OneToOne(mappedBy = "patient", cascade = CascadeType.ALL)`)
3. Speichere einen Patienten mit Adresse
4. Rufe den Patienten ab und gib Adresse aus

### UE_11.2_7: CascadeTypes vergleichen

Schreibe ein Programm `CascadeTest.java`:

1. Erstelle einen Patienten mit 2 Befunden und speichere ihn
2. Lösche den Patienten mit `em.remove()` – durch `CascadeType.ALL` sollten auch die Befunde gelöscht werden
3. Ändere `CascadeType.ALL` auf `CascadeType.PERSIST` – nun sollte das Löschen fehlschlagen
4. Erkläre in Kommentaren das Verhalten

### UE_11.2_8: Befund mit Verweis auf Patienten speichern

Schreibe ein Programm `BefundSpeichern.java`:

1. Rufe einen vorhandenen Patienten ab
2. Erstelle einen neuen Befund und setze den Patienten
3. Speichere den Befund mit `em.persist(befund)` (ohne Cascade)
4. Gib den Befund mit Patientennamen aus
