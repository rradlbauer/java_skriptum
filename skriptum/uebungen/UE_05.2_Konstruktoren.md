### UE_05.2_1: Patient mit Konstruktor

Erstelle eine Klasse `Patient` mit:

- Privaten Attributen: `vorname`, `nachname`, `geburtsdatum` (String), `diagnose`
- Einem Konstruktor, der alle Attribute initialisiert
- Einem parameterlosen Konstruktor, der Default-Werte setzt (z. B. "Unbekannt")
- Gettern und Settern für alle Attribute

In einer `Main`-Klasse:
1. Erzeuge einen Patienten mit vollständigen Daten
2. Erzeuge einen Patienten ohne Angaben (parameterloser Konstruktor)
3. Gib beide mit einer `ausgabe()`-Methode aus

### UE_05.2_2: this vs. self

Gegeben ist folgender Java-Code:

```java
public class Medikament {
    private String name;
    private double dosierung;

    public Medikament(String name, double dosierung) {
        name = name;
        dosierung = dosierung;
    }

    public void ausgabe() {
        System.out.println(name + ": " + dosierung + " mg");
    }

    public static void main(String[] args) {
        Medikament m = new Medikament("Ibuprofen", 400.0);
        m.ausgabe();
    }
}
```

1. Was wird dieses Programm ausgeben? (Teste es!)
2. Korrigiere den Konstruktor, sodass die Attribute korrekt gesetzt werden
3. Erkläre den Fehler

### UE_05.2_3: Befund-Klasse mit Validierung

Erstelle eine Klasse `Befund` im Package `at.htl.medizin` mit:

- Privaten Attributen: `patientName`, `befundDatum`, `diagnose`, `laborwert` (double)
- Parametrisiertem Konstruktor
- Im Konstruktor: Wenn `laborwert` negativ ist, setze ihn auf 0.0 und gib eine Warnung aus
- Gettern für alle Attribute
- Setter für `laborwert` mit Validierung (nicht negativ)
