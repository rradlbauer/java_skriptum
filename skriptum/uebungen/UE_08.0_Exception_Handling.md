### UE_08.0_1: NumberFormatException abfangen

Schreibe ein Programm `EingabeTest.java`, das:

1. Drei Strings mit Patienten-Altern definiert: `"25"`, `"abc"`, `"30"`
2. In einer Schleife versucht, jeden String mit `Integer.parseInt()` in eine Zahl umzuwandeln
3. Fängt `NumberFormatException` ab und gibt eine Fehlermeldung aus
4. Gibt alle erfolgreich geparsten Alter aus

### UE_08.0_2: Patienten-Alter validieren

Erstelle eine Klasse `Patient` mit einem Setter für `alter`:

```java
public void setAlter(int alter) throws IllegalArgumentException {
    if (alter < 0 || alter > 150) {
        throw new IllegalArgumentException("Ungültiges Alter: " + alter);
    }
    this.alter = alter;
}
```

In einer `main`-Methode:
1. Erzeuge einen Patienten
2. Versuche, das Alter auf -5 zu setzen (fängt die Exception)
3. Versuche, das Alter auf 200 zu setzen
4. Setze ein gültiges Alter (25)
5. Gib aus: "Patient alter: 25"

### UE_08.0_3: try-catch-finally

Schreibe ein Programm, das zeigt, dass der `finally`-Block immer ausgeführt wird:

```java
public class FinallyTest {
    public static void main(String[] args) {
        try {
            System.out.println("Im try-Block");
            int ergebnis = 10 / 0;  // ArithmeticException
            System.out.println("Nach der Division"); // Wird nicht erreicht
        } catch (ArithmeticException e) {
            System.out.println("Fehler: Division durch Null!");
        } finally {
            System.out.println("finally: Wird immer ausgeführt!");
        }
        System.out.println("Programm wird fortgesetzt...");
    }
}
```

Erweitere das Programm: Füge einen zweiten `try-catch`-Block hinzu, der eine `NullPointerException` auslöst und ebenfalls einen `finally`-Block hat.

### UE_08.0_4: try-with-resources

Schreibe ein Programm, das `try-with-resources` verwendet:

```java
import java.util.Scanner;
import java.io.File;
import java.io.FileNotFoundException;

public class TryWithResourcesTest {
    public static void main(String[] args) {
        // Verwende try-with-resources
        try (Scanner sc = new Scanner(new File("test.txt"))) {
            while (sc.hasNextLine()) {
                System.out.println(sc.nextLine());
            }
        } catch (FileNotFoundException e) {
            System.out.println("Datei nicht gefunden: " + e.getMessage());
        }
    }
}
```

1. Erstelle eine Datei `test.txt` mit einigen Patientendaten (Name, Alter, Diagnose pro Zeile)
2. Führe das Programm aus – es sollte funktionieren
3. Lösche oder verschiebe die Datei und führe das Programm erneut aus – die Exception sollte abgefangen werden

### UE_08.0_5: Eigene Exception für Blutdruck

Erstelle eine eigene Exception-Klasse:

```java
public class BlutdruckException extends Exception {
    public BlutdruckException(String message) {
        super(message);
    }
}
```

Erstelle eine Klasse `Blutdruckmessung` mit einer Methode:

```java
public void validiereBlutdruck(int systolisch, int diastolisch) throws BlutdruckException {
    if (systolisch < 60 || systolisch > 300) {
        throw new BlutdruckException("Ungültiger systolischer Wert: " + systolisch);
    }
    if (diastolisch < 30 || diastolisch > 200) {
        throw new BlutdruckException("Ungültiger diastolischer Wert: " + diastolisch);
    }
    System.out.println("Blutdruck " + systolisch + "/" + diastolisch + " ist gültig.");
}
```

In der `main`-Methode:
1. Rufe `validiereBlutdruck(140, 90)` auf – sollte OK sein
2. Rufe `validiereBlutdruck(20, 80)` auf – sollte Exception auslösen und abgefangen werden
3. Rufe `validiereBlutdruck(120, 250)` auf – sollte Exception auslösen
