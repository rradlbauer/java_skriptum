# 05.3 Methoden überladen (Overloading)

**Methodenüberladung** bedeutet, dass mehrere Methoden **denselben Namen** haben, aber **unterschiedliche Parameter** (Typ, Anzahl oder Reihenfolge). In Python ist das nicht direkt möglich – dort brauchst du Standardparameter oder `*args`.

## Beispiel: Dosierungsberechnung

```java
public class Dosierungsrechner {

    // Einfache Dosierung nach Gewicht
    public double berechneDosis(double gewicht) {
        return gewicht * 0.02;  // 2% des Körpergewichts
    }

    // Dosierung mit benutzerdefiniertem Faktor
    public double berechneDosis(double gewicht, double faktor) {
        return gewicht * faktor;
    }

    // Dosierung nach Alter und Gewicht (für Kinder)
    public double berechneDosis(int alter, double gewicht) {
        if (alter < 12) {
            return gewicht * 0.015;  // reduzierte Dosis
        } else {
            return gewicht * 0.02;
        }
    }
}
```

Verwendung:

```java
Dosierungsrechner rechner = new Dosierungsrechner();

double d1 = rechner.berechneDosis(70.0);          // 1. Methode
double d2 = rechner.berechneDosis(70.0, 0.025);   // 2. Methode
double d3 = rechner.berechneDosis(10, 30.0);      // 3. Methode (int, double)
```

## Wie erkennt Java die richtige Methode?

Der Compiler entscheidet anhand der **Parameterliste** (Anzahl und Typ), welche Methode aufgerufen wird. Das passiert bereits zur **Compile-Zeit**.

```java
berechneDosis(70.0)         // → Methode 1 (double)
berechneDosis(70.0, 0.025)  // → Methode 2 (double, double)
berechneDosis(10, 30.0)     // → Methode 3 (int, double)
```

## Konstruktor-Überladung

Besonders nützlich ist das Überladen von Konstruktoren. So kannst du verschiedene Initialisierungsmöglichkeiten anbieten:

```java
public class Patient {
    private String name;
    private int alter;
    private double koerpergroesse;

    // 1. Konstruktor: alle Angaben
    public Patient(String name, int alter, double koerpergroesse) {
        this.name = name;
        this.alter = alter;
        this.koerpergroesse = koerpergroesse;
    }

    // 2. Konstruktor: nur Name (andere Werte mit Defaults)
    public Patient(String name) {
        this.name = name;
        this.alter = 0;
        this.koerpergroesse = 0.0;
    }

    // 3. Konstruktor: Name und Alter
    public Patient(String name, int alter) {
        this.name = name;
        this.alter = alter;
        this.koerpergroesse = 0.0;
    }
}
```

## this() – Konstruktor von Konstruktor aufrufen

Mit `this(...)` kann ein Konstruktor einen anderen Konstruktor derselben Klasse aufrufen – das vermeidet Code-Wiederholung:

```java
public class Patient {
    private String name;
    private int alter;
    private double koerpergroesse;

    // Hauptkonstruktor
    public Patient(String name, int alter, double koerpergroesse) {
        this.name = name;
        this.alter = alter;
        this.koerpergroesse = koerpergroesse;
    }

    // Ruft Hauptkonstruktor mit Default-Werten auf
    public Patient(String name) {
        this(name, 0, 0.0);  // this() muss die ERSTE Anweisung sein
    }

    public Patient(String name, int alter) {
        this(name, alter, 0.0);
    }
}
```

## Vergleich mit Python

In Python kannst du nicht mehrere Methoden mit demselben Namen definieren – die letzte Definition gewinnt:

```python
# Python: nicht möglich
class Dosierung:
    def berechne(self, gewicht):      # wird überschrieben
        return gewicht * 0.02

    def berechne(self, gewicht, faktor):  # überschreibt die erste
        return gewicht * faktor
```

Python verwendet stattdessen Standardparameter oder `*args`/`**kwargs`.
