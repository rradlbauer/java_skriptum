# 02.2 Datentypen: Primitive Typen und Klassen-Datentypen

Java unterscheidet zwischen **primitiven Datentypen** (Wertetypen) und **Klassen-Datentypen** (Referenztypen).

## Primitive Datentypen

Primitive Datentypen speichern **Werte direkt**. Es gibt acht primitive Datentypen in Java:

| Typ | Größe | Wertebereich | Beispiel |
|-----|-------|-------------|---------|
| `byte` | 8 Bit | -128 bis 127 | `byte b = 42;` |
| `short` | 16 Bit | -32.768 bis 32.767 | `short s = 1000;` |
| `int` | 32 Bit | -2³¹ bis 2³¹-1 | `int i = 42;` |
| `long` | 64 Bit | -2⁶³ bis 2⁶³-1 | `long l = 42L;` |
| `float` | 32 Bit | ±3,4E-38 bis ±3,4E+38 | `float f = 3.14f;` |
| `double` | 64 Bit | ±1,7E-308 bis ±1,7E+308 | `double d = 3.14;` |
| `char` | 16 Bit | Unicode-Zeichen | `char c = 'A';` |
| `boolean` | 1 Bit | true / false | `boolean b = true;` |

Im Alltag verwendest du meist `int`, `double`, `boolean` und `char`.

**Wichtig:** Anders als in Python gibt es in Java keinen `**`-Operator für Potenzen. Verwende `Math.pow(basis, exponent)`:

```java
double quadrat = Math.pow(2, 3);  // 8.0
```

## Klassen-Datentypen (Referenztypen)

Klassen-Datentypen speichern **Referenzen** (Adressen) auf Objekte. Der wichtigste Klassen-Datentyp für den Anfang ist `String`:

```java
String name = "Anna";
String gruss = "Hallo";
String satz = name + " sagt " + gruss;  // Verkettung mit +
```

Andere Beispiele für Klassen-Datentypen:
- `String`
- Arrays (`int[]`, `String[]`)
- Alle selbst definierten Klassen (z. B. `Patient`, `Arzt`)
- Wrapper-Klassen (siehe unten)

## Wrapper-Klassen

Zu jedem primitiven Datentyp gibt es eine **Wrapper-Klasse**, die den Wert in ein Objekt "verpackt":

| Primitiver Typ | Wrapper-Klasse |
|---------------|---------------|
| `int` | `Integer` |
| `double` | `Double` |
| `boolean` | `Boolean` |
| `char` | `Character` |
| `long` | `Long` |
| `float` | `Float` |
| `byte` | `Byte` |
| `short` | `Short` |

### Autoboxing und Unboxing

Seit Java 5 erfolgt die Umwandlung zwischen primitiven Typen und ihren Wrapper-Klassen automatisch:

```java
int zahl = 42;
Integer wrapper = zahl;       // Autoboxing: int → Integer
int wiederPrimitiv = wrapper; // Unboxing: Integer → int
```

### Wozu Wrapper-Klassen?

- In Collections (wie `ArrayList`) können nur Objekte gespeichert werden, keine primitiven Typen
- Wrapper-Klassen bieten nützliche Methoden:

```java
int zahl = Integer.parseInt("42");     // String → int
String text = Integer.toString(42);    // int → String
boolean istZiffer = Character.isDigit('5');  // true
```

## Typumwandlung (Casting)

### Automatische Umwandlung (Widening)
Kleinere Datentypen werden automatisch in größere umgewandelt:

```java
int i = 100;
double d = i;  // automatisch: int → double, d = 100.0
```

### Explizites Casting (Narrowing)
Größere Datentypen müssen explizit gecastet werden:

```java
double d = 3.99;
int i = (int) d;  // i = 3 (Nachkommastellen werden abgeschnitten)
```

## Vergleich: Python vs. Java

```python
# Python
x = 42              # kein Typ nötig
y = "Hallo"         # dynamischer Typ
x = 3.14            # Typwechsel möglich
```

```java
// Java
int x = 42;         // Typ muss angegeben werden
String y = "Hallo";
// x = 3.14;        // Compiler-Fehler! double kann nicht int zugewiesen werden
```
