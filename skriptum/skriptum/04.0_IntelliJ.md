# 04.0 Einführung in IntelliJ

IntelliJ ist die empfohlene IDE für Java-Entwicklung in diesem Kurs – vergleichbar mit PyCharm, das du aus dem Python-Teil kennst (beide stammen von JetBrains).

## Projekt anlegen

1. **IntelliJ starten**
2. **"New Project"** auswählen
3. Folgende Einstellungen vornehmen:

| Feld | Wert |
|------|------|
| Name | `mein-java-projekt` |
| Location | Dein Projektordner |
| Language | Java |
| Build system | **IntelliJ** (kein Maven/Gradle für den Anfang) |
| JDK | Dein installiertes JDK auswählen (z. B. JDK 21) |
| Project template | **Command Line App** (oder "Generate project from template" deaktivieren) |

4. **"Create"** klicken

## Projektstruktur in IntelliJ

```
mein-java-projekt/
├── src/
│   └── (Package-Ordnerstruktur)
└── .idea/            (IntelliJ-Konfiguration, nicht verändern)
```

## Hello World mit IntelliJ

1. Rechtsklick auf `src` → **New → Java Class**
2. Name: `HelloWorld`
3. Code eingeben:

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hallo, IntelliJ!");
    }
}
```

4. Grüner Pfeil (▶) links neben der Klasse oder Methode klicken → **Run 'HelloWorld.main()'**
5. Oder: `Strg + Umschalt + F10` (Windows/Linux) / `Cmd + Umschalt + R` (Mac)

Die Ausgabe erscheint im **Run**-Fenster unten.

## Wichtige Einstellungen

### Schriftart vergrößern
`Datei → Settings (Strg+Alt+S)` → `Editor → Font` → Größe anpassen

Alternative: `Datei → Settings → Editor → General → Mouse Control` → "Change font size with Ctrl+Mouse Wheel" aktivieren

### Zeilennummern
`Datei → Settings → Editor → General → Appearance` → "Show line numbers" (standardmäßig aktiviert)

### Auto-Import
`Datei → Settings → Editor → General → Auto Import` → "Add unambiguous imports on the fly" aktivieren

### Theme
`Datei → Settings → Appearance & Behavior → Appearance` → Theme: IntelliJ Light / Darcula

## Wichtige Shortcuts

| Shortcut | Funktion |
|----------|----------|
| `Strg + Leertaste` | Code-Vervollständigung |
| `Strg + Shift + Leertaste` | Smart Completion (typgerechte Vorschläge) |
| `Strg + P` | Parameter-Info |
| `Strg + B` | Zur Deklaration springen |
| `Strg + Alt + V` | Variable extrahieren |
| `Alt + Enter` | Schnellkorrektur / Vorschläge |
| `Strg + /` | Zeile aus-/einkommentieren |
| `Strg + Shift + F10` | Aktuelle Datei ausführen |
| `Shift + F10` | Letzte Ausführung wiederholen |
| `Strg + D` | Zeile duplizieren |
| `Strg + Alt + L` | Code formatieren |

## Debugger in IntelliJ

Der Debugger funktioniert ähnlich wie in PyCharm:

1. Klicke links neben die Zeilennummer, um einen **Breakpoint** zu setzen (roter Punkt)
2. Führe das Programm mit dem **Debug**-Symbol (Käfer) aus oder `Umschalt + F9`
3. Steuerung:

| Taste | Funktion |
|-------|----------|
| `F8` | Step Over (nächste Zeile) |
| `F7` | Step Into (in Methode hinein) |
| `Shift + F8` | Step Out (Methode verlassen) |
| `F9` | Resume (weiter bis nächster Breakpoint) |

## Häufige Probleme und Lösungen

| Problem | Lösung |
|---------|--------|
| "JDK not found" | In Project Structure (`Strg+Alt+Umschalt+S`) das JDK setzen |
| "Class not found" | Prüfen, ob der Package-Pfad stimmt |
| Rote Unterstreichungen | `Alt + Enter` für Korrekturvorschläge |
| Unübersichtliches Layout | `Window → Restore Default Layout` |
