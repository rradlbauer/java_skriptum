# 09.3 Lambda-Ausdrücke

**Lambda-Ausdrücke** (Java 8+) sind eine kompakte Schreibweise für anonyme Funktionen. Sie sind die Grundlage für die funktionale Programmierung in Java und Voraussetzung für die Streaming API.

## Was ist ein Lambda?

Ein Lambda-Ausdruck ist eine **anonyme Methode** – eine Methode ohne Namen, die du direkt an Ort und Stelle definierst:

```java
// Lambda: nimm eine Zahl, gib sie verdoppelt zurück
(int x) -> { return x * 2; }

// Kurzform (wenn nur ein Ausdruck)
(x) -> x * 2

// Noch kürzer (bei einem Parameter ohne Typ)
x -> x * 2
```

Vergleich mit Python:

```python
# Python
lambda x: x * 2
```

## Syntax

```
(Parameterliste) -> { Methodenrumpf }
   (int a, int b)  ->  { return a + b; }
```

Varianten:

| Form | Beispiel | Beschreibung |
|------|----------|-------------|
| Kein Parameter | `() -> System.out.println("Hallo")` | Nur `()` |
| Ein Parameter | `x -> x * 2` | Klammern optional |
| Mehrere Parameter | `(a, b) -> a + b` | Klammern nötig |
| Typangabe | `(int a, int b) -> a + b` | Typen optional |
| Mehrere Anweisungen | `(a, b) -> { int s = a + b; return s; }` | Geschweifte Klammern + `return` |

## Funktionale Interfaces

Ein Lambda kann immer dort verwendet werden, wo ein **funktionales Interface** erwartet wird – ein Interface mit genau **einer abstrakten Methode**:

```java
@FunctionalInterface
interface Berechnung {
    int berechne(int a, int b);  // genau eine abstrakte Methode
}
```

Verwendung mit Lambda:

```java
Berechnung add = (a, b) -> a + b;
Berechnung sub = (a, b) -> a - b;

System.out.println(add.berechne(3, 4));  // 7
System.out.println(sub.berechne(3, 4));  // -1
```

### Wichtige built-in funktionale Interfaces

Java liefert viele funktionale Interfaces im Paket `java.util.function`:

| Interface | Methode | Beschreibung | Beispiel-Lambda |
|-----------|---------|-------------|-----------------|
| `Predicate<T>` | `boolean test(T)` | Prüft eine Bedingung | `s -> s.length() > 3` |
| `Consumer<T>` | `void accept(T)` | Führt eine Aktion aus | `s -> System.out.println(s)` |
| `Function<T,R>` | `R apply(T)` | Wandelt T in R um | `s -> s.toUpperCase()` |
| `Supplier<T>` | `T get()` | Liefert einen Wert | `() -> new Patient()` |
| `BinaryOperator<T>` | `T apply(T,T)` | Kombiniert zwei Werte | `(a, b) -> a + b` |
| `Comparator<T>` | `int compare(T,T)` | Vergleicht zwei Werte | `(a, b) -> a - b` |

### Predicate – Bedingung prüfen

```java
import java.util.function.Predicate;

Predicate<Integer> isGerade = n -> n % 2 == 0;
Predicate<String> istLang = s -> s.length() > 5;

System.out.println(isGerade.test(4));   // true
System.out.println(istLang.test("Hi")); // false
```

### Consumer – Aktion ausführen

```java
import java.util.function.Consumer;

Consumer<String> drucken = s -> System.out.println("Patient: " + s);
drucken.accept("Anna Müller");  // Patient: Anna Müller
```

### Function – umwandeln

```java
import java.util.function.Function;

Function<String, Integer> laenge = s -> s.length();
Function<String, String> gross = s -> s.toUpperCase();

System.out.println(laenge.apply("Anna"));  // 4
System.out.println(gross.apply("anna"));   // ANNA
```

### Supplier – Wert liefern

```java
import java.util.function.Supplier;

Supplier<Patient> erstellePatient = () -> new Patient("Unbekannt", 0, "");
Patient p = erstellePatient.get();
```

## Lambda mit Collections

Die Collections-API verwendet Lambda-Ausdrücke in vielen Methoden:

### forEach

```java
import java.util.List;

List<Patient> patienten = List.of(
    new Patient("Anna", "Grippe", 25),
    new Patient("Ben", "Diabetes", 30)
);

// Mit Lambda
patienten.forEach(p -> System.out.println(p.getName()));

// Mit Methoden-Referenz (kürzer)
patienten.forEach(System.out::println);
```

### removeIf

```java
import java.util.ArrayList;

List<Patient> patienten = new ArrayList<>();
patienten.add(new Patient("Anna", "Grippe", 25));
patienten.add(new Patient("Ben", "Diabetes", 30));
patienten.add(new Patient("Clara", "Grippe", 22));

patienten.removeIf(p -> p.getDiagnose().equals("Grippe"));
// Nur noch Ben bleibt in der Liste
```

### sort mit Comparator

```java
import java.util.Comparator;

List<Patient> patienten = new ArrayList<>();
// ... Patienten hinzufügen

// Lambda als Comparator
patienten.sort((p1, p2) -> p1.getAlter() - p2.getAlter());
patienten.sort((p1, p2) -> p1.getName().compareTo(p2.getName()));

// Mit Comparator-Hilfsmethoden (kürzer)
patienten.sort(Comparator.comparing(Patient::getAlter));
patienten.sort(Comparator.comparing(Patient::getName));
```

## Variablen in Lambdas (Capture)

Ein Lambda kann auf Variablen der umgebenden Methode zugreifen – sie müssen **effectively final** sein (dürfen nicht mehr geändert werden):

```java
String prefix = "Dr. ";  // effectively final (wird nicht geändert)

List<String> namen = List.of("Anna", "Ben");
namen.forEach(name -> System.out.println(prefix + name));
// Ausgabe: Dr. Anna, Dr. Ben

// prefix = "Prof. ";  // Fehler! Variable ist nicht mehr effectively final
```

## Lambda vs. Anonyme Klasse

Vor Java 8 wurden anonyme innere Klassen verwendet. Lambdas sind kürzer:

```java
// Anonyme Klasse (alt)
patienten.sort(new Comparator<Patient>() {
    @Override
    public int compare(Patient p1, Patient p2) {
        return p1.getAlter() - p2.getAlter();
    }
});

// Lambda (neu)
patienten.sort((p1, p2) -> p1.getAlter() - p2.getAlter());
```

## Methoden-Referenzen

Methoden-Referenzen sind eine noch kürzere Schreibweise, wenn ein Lambda nur eine vorhandene Methode aufruft:

```java
// Lambda
patienten.forEach(p -> System.out.println(p));

// Methoden-Referenz
patienten.forEach(System.out::println);
```

Die Form `Klasse::methode` wird im Kapitel zur Streaming API ausführlich verwendet.

## Zusammenfassung

### Wichtige Begriffe
- Lambda-Ausdruck `(x -> ...)`
- Funktionales Interface (`@FunctionalInterface`)
- `Predicate<T>` / `Consumer<T>` / `Function<T,R>` / `Supplier<T>`
- `forEach` / `removeIf` / `sort`
- Effectively final (Variable Capture)
- Methoden-Referenz `::`

### Das sollst du können
- Lambda-Ausdrücke in verschiedenen Formen schreiben
- Funktionale Interfaces erkennen und verwenden
- `Predicate`, `Consumer`, `Function`, `Supplier` anwenden
- Lambda mit Collection-Methoden (`forEach`, `removeIf`, `sort`) nutzen
- Den Unterschied zwischen Lambda und anonymer Klasse erklären
- Methoden-Referenzen als Kurzform von Lambdas verwenden
