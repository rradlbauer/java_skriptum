# 08.0 Exception Handling in Java

Fehlerbehandlung mit Exceptions kennst du aus Python (`try/except`). Java hat ein ähnliches, aber umfangreicheres System.

## Konzept der Fehlerbehandlung mit Exceptions

Eine **Exception** ist ein Ereignis, das den normalen Programmablauf unterbricht. In Python wie in Java werden Exceptions ausgelöst (`raise`/`throw`) und können abgefangen werden (`except`/`catch`).

## Unterschiede zu Python

| Aspekt | Python | Java |
|--------|--------|------|
| Auslösen | `raise Exception("msg")` | `throw new Exception("msg")` |
| Abfangen | `try/except` | `try/catch` |
| Immer ausführen | `finally` | `finally` |
| Exception-Typ muss deklariert werden? | Nein | Ja (checked exceptions) |

Der wichtigste Unterschied: Java unterscheidet zwischen **checked** und **unchecked Exceptions**.

## Checked vs. Unchecked Exceptions

### Unchecked Exceptions (RuntimeException)
- Müssen **nicht** deklariert oder abgefangen werden
- Sind Programmierfehler (z. B. Division durch Null, NullPointerException)
- Entsprechen den meisten Python-Exceptions

```java
int[] zahlen = {1, 2, 3};
System.out.println(zahlen[10]);  // ArrayIndexOutOfBoundsException (unchecked)
```

### Checked Exceptions
- **Müssen** deklariert (`throws`) oder abgefangen (`catch`) werden
- Sind vorhersehbare Fehler (z. B. Datei nicht gefunden, Netzwerkfehler)
- Der Compiler erzwingt die Behandlung

```java
// Compiler-Fehler! IOException muss behandelt werden
try {
    Thread.sleep(1000);  // InterruptedException (checked)
} catch (InterruptedException e) {
    System.out.println("Warten wurde unterbrochen");
}
```

## try/catch/finally

```java
public class PatientVerwaltung {
    public static void main(String[] args) {
        try {
            int alter = Integer.parseInt("abc");  // löst NumberFormatException aus
            System.out.println("Alter: " + alter);
        } catch (NumberFormatException e) {
            System.out.println("Fehler: Keine gültige Zahl!");
            System.out.println("Details: " + e.getMessage());
        } finally {
            System.out.println("Wird immer ausgeführt.");
        }
    }
}
```

### Mehrere catch-Blöcke

Du kannst verschiedene Exception-Typen unterschiedlich behandeln:

```java
try {
            String eingabe = "abc";
            int zahl = Integer.parseInt(eingabe);
            int ergebnis = 100 / zahl;
        } catch (NumberFormatException e) {
            System.out.println("Keine gültige Zahl: " + e.getMessage());
        } catch (ArithmeticException e) {
            System.out.println("Division durch Null!");
        } catch (Exception e) {
            System.out.println("Allgemeiner Fehler: " + e.getMessage());
        }
```

**Wichtig:** Spezifischere Exceptions müssen vor allgemeineren kommen.

### try-with-resources (Java 7+)

Für Ressourcen wie Dateien oder Datenbankverbindungen gibt es `try-with-resources`. Die Ressource wird automatisch geschlossen, auch wenn eine Exception auftritt:

```java
import java.util.Scanner;
import java.io.File;

try (Scanner sc = new Scanner(new File("patienten.csv"))) {
    while (sc.hasNextLine()) {
        System.out.println(sc.nextLine());
    }
} catch (java.io.FileNotFoundException e) {
    System.out.println("Datei nicht gefunden!");
}
```

Die Ressource muss `AutoCloseable` implementieren (wie `Scanner` oder `BufferedReader`).

Vergleich mit Python:

```python
# Python
with open("patienten.csv") as f:
    for line in f:
        print(line)
```

Beide Konzepte (`with` in Python, `try-with-resources` in Java) stellen sicher, dass Ressourcen ordnungsgemäß geschlossen werden.

## throw – Eigene Exceptions auslösen

```java
public void setAlter(int alter) {
    if (alter < 0 || alter > 150) {
        throw new IllegalArgumentException("Ungültiges Alter: " + alter);
    }
    this.alter = alter;
}
```

### throws – Exception deklarieren

Wenn eine Methode eine checked Exception nicht selbst behandelt, muss sie diese deklarieren:

```java
public void dateiLesen(String pfad) throws java.io.IOException {
    // Methode behandelt IOException nicht selbst
    // Der Aufrufer muss sie abfangen oder ebenfalls deklarieren
}

public void verarbeiteDatei(String pfad) {
    try {
        dateiLesen(pfad);
    } catch (java.io.IOException e) {
        System.out.println("Fehler beim Lesen der Datei");
    }
}
```

## Eigene Exception-Klassen

Du kannst eigene Exception-Klassen definieren:

```java
public class UngueltigePatientendatenException extends Exception {
    public UngueltigePatientendatenException(String message) {
        super(message);
    }
}

// Verwendung
public void validierePatient(String name, int alter)
        throws UngueltigePatientendatenException {
    if (name == null || name.isEmpty()) {
        throw new UngueltigePatientendatenException("Name darf nicht leer sein");
    }
    if (alter < 0 || alter > 150) {
        throw new UngueltigePatientendatenException("Ungültiges Alter: " + alter);
    }
}
```

## Zusammenfassung

### Wichtige Begriffe
- `try` / `catch` / `finally`
- `throw` / `throws`
- Checked / Unchecked Exception
- `try-with-resources`
- `AutoCloseable`
- Exception-Hierarchie (`Throwable` → `Exception` / `Error`)

### Das sollst du können
- Exceptions mit `try/catch` abfangen
- Eigene Exceptions mit `throw` auslösen
- Checked vs. Unchecked Exceptions unterscheiden
- `try-with-resources` für automatische Ressourcenverwaltung verwenden
- Eigene Exception-Klassen definieren
