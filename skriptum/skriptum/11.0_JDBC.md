# 11.0 Datenbankzugriff mit JDBC

**JDBC** (Java Database Connectivity) ist die Standard-API für den Zugriff auf relationale Datenbanken aus Java. Du kannst SQL-Abfragen ausführen, Daten lesen, schreiben und aktualisieren.

## Überblick

JDBC besteht aus wenigen zentralen Klassen und Interfaces:

| Klasse/Interface | Aufgabe |
|-----------------|---------|
| `DriverManager` | Verwaltet Datenbanktreiber, erstellt Verbindungen |
| `Connection` | Repräsentiert eine Datenbankverbindung |
| `PreparedStatement` | Führt SQL mit Parameter-Binding aus (sicher vor SQL-Injection) |
| `ResultSet` | Enthält die Ergebnismenge einer Abfrage |
| `SQLException` | Fehlerbehandlung bei Datenbankzugriffen |

### Ablauf einer JDBC-Operation

```
Connection herstellen (URL, Benutzer, Passwort)
    → PreparedStatement erzeugen (SQL mit ?-Platzhaltern)
        → Parameter setzen (setString, setInt, ...)
            → Query ausführen (executeQuery / executeUpdate)
                → Ergebnis verarbeiten (ResultSet)
                    → Verbindung schließen
```

## Voraussetzung: MySQL-Treiber

Für den Zugriff auf MySQL benötigst du den Connector/J-Treiber. Lade `mysql-connector-j-XXX.jar` von [https://dev.mysql.com/downloads/connector/j/](https://dev.mysql.com/downloads/connector/j/) herunter und binde ihn in dein Projekt ein (in IntelliJ: `File → Project Structure → Libraries → + → Java → JAR auswählen`).

## Verbindungsaufbau

```java
import java.sql.*;

public class VerbindungsTest {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/medizin";
        String user = "root";
        String password = "passwort";

        try (Connection conn = DriverManager.getConnection(url, user, password)) {
            System.out.println("Verbindung hergestellt: " + conn.getCatalog());
        } catch (SQLException e) {
            System.err.println("Fehler: " + e.getMessage());
        }
    }
}
```

Die URL setzt sich zusammen aus: `jdbc:mysql://HOST:PORT/DATENBANK`

## Query – Liste von Objekten erstellen

Nehmen wir eine Tabelle `patient` an:

```sql
CREATE TABLE patient (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    alter INT NOT NULL,
    diagnose VARCHAR(200)
);
```

### Daten abfragen und in Objekte umwandeln

```java
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

public class Patient {
    private int id;
    private String name;
    private int alter;
    private String diagnose;

    // Konstruktor, Getter, Setter, toString ...
    public Patient(int id, String name, int alter, String diagnose) {
        this.id = id; this.name = name; this.alter = alter; this.diagnose = diagnose;
    }

    @Override
    public String toString() {
        return id + ": " + name + " (" + alter + ", " + diagnose + ")";
    }
}
```

```java
public class PatientDAO {
    public List<Patient> allePatienten() throws SQLException {
        String sql = "SELECT id, name, alter, diagnose FROM patient";
        List<Patient> liste = new ArrayList<>();

        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement stmt = conn.prepareStatement(sql);
             ResultSet rs = stmt.executeQuery()) {

            while (rs.next()) {
                Patient p = new Patient(
                    rs.getInt("id"),
                    rs.getString("name"),
                    rs.getInt("alter"),
                    rs.getString("diagnose")
                );
                liste.add(p);
            }
        }
        return liste;
    }
}
```

Wichtig: `Connection`, `PreparedStatement` und `ResultSet` werden in `try-with-resources` verwendet – sie schließen sich automatisch.

## Prepared Statements mit Parameter-Binding

Statt SQL-Zeichenketten zu concatenaten (gefährlich! SQL-Injection!), verwendest du `?`-Platzhalter:

```java
public List<Patient> sucheNachDiagnose(String diagnose) throws SQLException {
    String sql = "SELECT id, name, alter, diagnose FROM patient WHERE diagnose = ?";

    try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
         PreparedStatement stmt = conn.prepareStatement(sql)) {

        stmt.setString(1, diagnose);  // 1 = erster Platzhalter

        try (ResultSet rs = stmt.executeQuery()) {
            List<Patient> liste = new ArrayList<>();
            while (rs.next()) {
                liste.add(new Patient(
                    rs.getInt("id"), rs.getString("name"),
                    rs.getInt("alter"), rs.getString("diagnose")
                ));
            }
            return liste;
        }
    }
}
```

Vergleich mit Python:

```python
# Python (dangerous): cursor.execute(f"SELECT * FROM patient WHERE diagnose = '{d}'")
# Python (safe):      cursor.execute("SELECT * FROM patient WHERE diagnose = ?", (d,))
```

## Objekt in Tabelle speichern (INSERT)

```java
public void einfuegen(Patient p) throws SQLException {
    String sql = "INSERT INTO patient (name, alter, diagnose) VALUES (?, ?, ?)";

    try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
         PreparedStatement stmt = conn.prepareStatement(sql)) {

        stmt.setString(1, p.getName());
        stmt.setInt(2, p.getAlter());
        stmt.setString(3, p.getDiagnose());
        stmt.executeUpdate();
    }
}
```

## Auto-increment Key auslesen

Nach einem INSERT möchtest du oft die generierte ID zurückhaben:

```java
public Patient einfuegenMitId(Patient p) throws SQLException {
    String sql = "INSERT INTO patient (name, alter, diagnose) VALUES (?, ?, ?)";

    try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
         PreparedStatement stmt = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)) {

        stmt.setString(1, p.getName());
        stmt.setInt(2, p.getAlter());
        stmt.setString(3, p.getDiagnose());
        stmt.executeUpdate();

        try (ResultSet keys = stmt.getGeneratedKeys()) {
            if (keys.next()) {
                p.setId(keys.getInt(1));  // Objekt mit generierter ID aktualisieren
            }
        }
        return p;
    }
}
```

Der zweite Parameter `Statement.RETURN_GENERATED_KEYS` teilt JDBC mit, dass du die automatisch generierten Schlüssel abrufen möchtest.

## Update durchführen

```java
public void aktualisiereDiagnose(int id, String neueDiagnose) throws SQLException {
    String sql = "UPDATE patient SET diagnose = ? WHERE id = ?";

    try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
         PreparedStatement stmt = conn.prepareStatement(sql)) {

        stmt.setString(1, neueDiagnose);
        stmt.setInt(2, id);

        int betroffen = stmt.executeUpdate();
        System.out.println(betroffen + " Zeilen aktualisiert.");
    }
}
```

## Zusammenfassung

### Wichtige Begriffe
- JDBC (Java Database Connectivity)
- `DriverManager` / `Connection`
- `PreparedStatement`
- `ResultSet`
- `executeQuery()` / `executeUpdate()`
- `RETURN_GENERATED_KEYS`
- Parameter-Binding mit `?`
- `try-with-resources` für Datenbankressourcen

### Das sollst du können
- Eine JDBC-Verbindung zu MySQL aufbauen
- SQL-Abfragen mit `PreparedStatement` ausführen
- Parameter mit `setString()`, `setInt()` binden
- Ergebniszeilen mit `ResultSet` in Objekte umwandeln
- INSERT mit auto-increment Key durchführen und die ID auslesen
- UPDATE- und DELETE-Operationen ausführen
- Den Unterschied zwischen sicherer (PreparedStatement) und unsicherer (String-Concat) SQL-Erstellung erklären
