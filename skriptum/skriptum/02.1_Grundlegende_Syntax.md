# 02.1 Grundlegende Syntax und strenge Typisierung

Nach den ersten Experimenten mit JShell sehen wir uns die Java-Syntax nun systematisch an.

## Grundlegende Syntaxregeln

| Aspekt | Java | Python |
|--------|------|--------|
| **Statement-Ende** | `;` (Semikolon) | Zeilenumbruch |
| **Code-Blöcke** | `{ ... }` | Einrückung |
| **Variablen-Deklaration** | `Typ name = wert;` | `name = wert` |
| **Kommentare** | `//` und `/* */` | `#` |
| **Case-Sensitivity** | Groß-/Kleinschreibung wichtig | Groß-/Kleinschreibung wichtig |

## Beispiel: Blockstruktur

```java
if (temperatur > 37.5) {
    System.out.println("Fieber!");
    System.out.println("Bitte zum Arzt gehen.");
} else {
    System.out.println("Temperatur normal.");
}
```

```python
# Python-Äquivalent
if temperatur > 37.5:
    print("Fieber!")
    print("Bitte zum Arzt gehen.")
else:
    print("Temperatur normal.")
```

## Strenge Typisierung

Java ist **stark und statisch typisiert**. Jede Variable hat einen festen Typ, der bereits zur Compile-Zeit bekannt sein muss.

```java
int anzahl = 10;        // OK
anzahl = "zehn";        // Compiler-Fehler! String kann nicht int zugewiesen werden
```

In Python wäre das problemlos möglich:

```python
anzahl = 10             # OK
anzahl = "zehn"         # auch OK (dynamische Typisierung)
```

Die strenge Typisierung hat Vorteile:
- Viele Fehler werden bereits beim Kompilieren erkannt
- Der Code ist selbst-dokumentierend (Typen sind sichtbar)
- Bessere Unterstützung durch die IDE (Autovervollständigung, Refactoring)

## Deklaration und Initialisierung

```java
int alter;                  // Deklaration (noch kein Wert)
alter = 25;                 // Initialisierung (Wert zuweisen)

double koerpergroesse = 1.75;  // Deklaration + Initialisierung

final double MWST = 0.20;   // Konstante (mit final, kann nicht geändert werden)
```

Das Schlüsselwort `final` macht eine Variable zu einer Konstanten – vergleichbar mit Konventionen in Python (GROSSBUCHSTABEN), aber vom Compiler erzwungen.

## Namenskonventionen

| Element | Konvention | Beispiel |
|---------|-----------|---------|
| Variablen | camelCase, beginnt mit Kleinbuchstabe | `patientName`, `koerperGroesse` |
| Methoden | camelCase, beginnt mit Kleinbuchstabe | `berechneBMI()`, `getPatient()` |
| Klassen | PascalCase, beginnt mit Großbuchstabe | `Patient`, `Krankenhaus` |
| Konstanten | UPPER_SNAKE_CASE | `MAX_PATIENTEN`, `PI` |
| Packages | lowercase, Punkte getrennt | `at.htl.medizin` |
