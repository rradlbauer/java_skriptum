# 06.0 Referenzen

Das Verständnis von Referenzen ist entscheidend, um Java (und OOP allgemein) zu beherrschen.

## Klassendatentypen vs. primitive Datentypen

Dies ist der fundamentale Unterschied:

**Primitive Datentypen** speichern den **Wert direkt**:

```java
int a = 10;
int b = a;  // b bekommt eine KOPIE von a (b = 10)
b = 20;     // a bleibt 10
```

```
a: [10]
b: [10] → [20]
```

**Klassendatentypen** speichern eine **Referenz (Adresse)** auf das Objekt:

```java
Patient p1 = new Patient("Anna");
Patient p2 = p1;      // p2 zeigt auf DASSELBE Objekt wie p1
p2.setName("Ben");    // Ändert das Objekt, auf das auch p1 zeigt
System.out.println(p1.getName());  // "Ben"
```

```
p1: [●] → Objekt: {name: "Anna"} → {name: "Ben"}
p2: [●] ┘
```

## Konsequenzen der Verwendung von Referenzen

### 1. Mehrere Variablen, ein Objekt

```java
Patient a = new Patient("Anna", 25);
Patient b = a;         // b ist eine zweite Referenz auf DASSELBE Objekt

b.setAlter(30);        // Änderung über b
System.out.println(a.getAlter());  // 30 – auch über a sichtbar!
```

### 2. Methoden-Parameter

Auch bei Methodenaufrufen werden Referenzen übergeben:

```java
public void aendereName(Patient p) {
    p.setName("Geändert");  // Ändert das Original-Objekt
}

Patient p = new Patient("Anna");
aendereName(p);
System.out.println(p.getName());  // "Geändert"
```

Bei primitiven Typen wird eine Kopie übergeben:

```java
public void aendereZahl(int x) {
    x = 100;  // Ändert nur die Kopie
}

int zahl = 5;
aendereZahl(zahl);
System.out.println(zahl);  // 5 (unverändert!)
```

### 3. null und NullPointerException

Eine Referenz, die auf kein Objekt zeigt, hat den Wert `null`:

```java
Patient p = null;      // p zeigt auf kein Objekt
p.setName("Anna");     // NullPointerException (Programm stürzt ab)
```

Vergleich mit Python: `None` in Python entspricht `null` in Java.

## Sinn von immutable Klassen

Eine **immutable** (unveränderliche) Klasse kann nach der Erstellung nicht mehr geändert werden. Das ist besonders wichtig bei Referenzen.

### Beispiele für immutable Klassen in Java

- `String`
- Wrapper-Klassen (`Integer`, `Double`, ...)
- `LocalDate`, `LocalTime`

### Warum ist String immutable?

```java
String s1 = "Hallo";
String s2 = s1;
s1 = s1 + " Welt";   // Es wird ein NEUES String-Objekt erzeugt

System.out.println(s2);  // "Hallo" (s2 zeigt immer noch auf das originale Objekt)
```

Wäre `String` veränderbar, würde `s2` plötzlich ebenfalls "Hallo Welt" anzeigen.

### Vorteile von immutable Klassen

- **Sicherheit:** Ein Objekt kann nicht unerwartet geändert werden
- **Einfache Nebenläufigkeit:** Immutable Objekte sind automatisch thread-sicher
- **Referenzielle Integrität:** Wenn mehrere Referenzen auf dasselbe Objekt zeigen, kann keine sie unbeabsichtigt ändern

### Eine eigene immutable Klasse schreiben

```java
public final class Befund {     // final verhindert Vererbung
    private final String diagnose;     // final = kann nicht mehr geändert werden
    private final double wert;

    public Befund(String diagnose, double wert) {
        this.diagnose = diagnose;
        this.wert = wert;
    }

    public String getDiagnose() { return diagnose; }
    public double getWert() { return wert; }
    // Keine Setter!
}
```
