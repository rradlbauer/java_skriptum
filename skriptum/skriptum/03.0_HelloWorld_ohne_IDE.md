# 03.0 Hello World Programm ohne Entwicklungsumgebung

Bevor wir IntelliJ verwenden, erstellen wir ein Java-Programm mit einem einfachen Texteditor und der Kommandozeile. So verstehst du, was im Hintergrund passiert.

## Das erste Java-Programm

Erstelle eine Datei namens `HelloWorld.java` mit folgendem Inhalt:

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hallo, Welt!");
    }
}
```

## JDK-Tools: javac und java

Das JDK stellt zwei wichtige Kommandozeilen-Werkzeuge bereit:

### 1. Compiler: `javac`

Der Compiler übersetzt die `.java`-Datei in Byte Code:

```
javac HelloWorld.java
```

Nach erfolgreicher Kompilierung findest du eine neue Datei: `HelloWorld.class`. Das ist der Byte Code.

```
HelloWorld.java  →  (javac)  →  HelloWorld.class
```

### 2. Ausführen: `java`

Die Laufzeitumgebung führt den Byte Code aus:

```
java HelloWorld
```

Ausgabe:
```
Hallo, Welt!
```

**Wichtig:** Beim Ausführen mit `java` wird **ohne** `.class`-Endung angegeben.

## Die main-Methode im Detail

Jedes ausführbare Java-Programm benötigt eine `main`-Methode mit genau dieser Signatur:

```java
public static void main(String[] args)
```

| Teil | Bedeutung |
|------|-----------|
| `public` | Die Methode ist von außen zugänglich |
| `static` | Die Methode gehört zur Klasse, nicht zu einer Instanz |
| `void` | Die Methode gibt keinen Wert zurück |
| `main` | Name der Methode (vom JVM vorgegeben) |
| `String[] args` | Kommandozeilen-Argumente (wie `sys.argv` in Python) |

## Klassenname = Dateiname

In Java muss der **Dateiname exakt dem Klassennamen** entsprechen:

| Klasse | Dateiname |
|--------|-----------|
| `public class HelloWorld` | `HelloWorld.java` |
| `public class Patient` | `Patient.java` |

Ein Unterschied zu Python: In Python kann eine Datei `mein_programm.py` mehrere Klassen enthalten, und der Dateiname ist frei wählbar.

## Alles muss in einer Klasse stehen

In Java muss **jeder Code innerhalb einer Klasse** stehen. Es gibt keine Möglichkeit, Code außerhalb einer Klasse zu schreiben – kein Skript-Modus wie in Python.

**Python** (erlaubt Code außerhalb von Klassen):

```python
# top-level Code, direkt in der Datei
print("Hallo Welt")

def funktion():
    pass

class MeineKlasse:
    pass
```

**Java** (alles in einer Klasse):

```java
public class HelloWorld {           // Alles in dieser Klasse
    public static void main(String[] args) {
        System.out.println("Hallo Welt");
    }
}
```

Das ist kein Zufall, sondern eine grundlegende Design-Entscheidung: Java ist **rein objektorientiert** – jedes Programm besteht aus Klassen, die zusammenarbeiten.

## Zusammenhang: Dateiname, Klassenname, Package und Verzeichnis

Für den Compiler hängen vier Dinge untrennbar zusammen:

```text
┌─────────────────────────────────────────────────────┐
│  Dateiname:   HelloWorld.java                       │
│  Klassenname: HelloWorld               ← muss       │
│  Package:     at.htl.medizin            überein-    │
│  Verzeichnis: src/at/htl/medizin/        stimmen    │
└─────────────────────────────────────────────────────┘
```

### Die drei Regeln

1. **Klassenname = Dateiname** – Die `public`-Klasse muss genauso heißen wie die `.java`-Datei
2. **Package = Verzeichnis** – Die `package`-Anweisung entspricht dem Ordner, in dem die Datei liegt. `package at.htl.medizin;` → Datei in `at/htl/medizin/`
3. **Alles wird vom Compiler geprüft** – Stimmt etwas nicht, gibt es einen Compiler-Fehler

### Vollständiges Beispiel

```text
Projektordner/
└── src/
    └── at/
        └── htl/
            └── medizin/
                └── Hallo.java        (package at.htl.medizin; public class Hallo { ... })
```

Kompilieren und ausführen:

```bash
javac -d bin src/at/htl/medizin/Hallo.java
java -cp bin at.htl.medizin.Hallo
```

> **Details zu Packages** findest du im nächsten Kapitel [03.1 Packages](03.1_Packages.md).

## Kompilierungsfehler kennenlernen

Ändere bewusst etwas am Code, um typische Fehlermeldungen zu sehen:

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hallo, Welt!")
        // Fehler: fehlendes Semikolon!
    }
}
```

Beim Kompilieren:

```
HelloWorld.java:3: error: ';' expected
    System.out.println("Hallo, Welt!")
                                       ^
1 error
```

Diese Fehler werden zur **Compile-Zeit** erkannt – ein Vorteil gegenüber Python, wo solche Fehler erst zur Laufzeit auftreten.

## Konsolen-Ein- und Ausgabe

### Text auf der Konsole ausgeben

`System.out.println()` gibt Text aus und fügt am Ende einen Zeilenumbruch an. `System.out.print()` gibt Text **ohne** Zeilenumbruch aus:

```java
System.out.println("Zeile 1");
System.out.print("Zeile 2 (ohne Umbruch)");
System.out.println(" – hier geht's weiter");

// Ausgabe:
// Zeile 1
// Zeile 2 (ohne Umbruch) – hier geht's weiter
```

Mit `System.out.printf()` kannst du formatierte Ausgaben erzeugen – ähnlich wie `f-Strings` in Python:

```java
String name = "Anna";
int alter = 25;
System.out.printf("Patient: %s, Alter: %d Jahre%n", name, alter);
// Ausgabe: Patient: Anna, Alter: 25 Jahre
```

| Platzhalter | Bedeutung |
|-------------|-----------|
| `%s` | String |
| `%d` | ganze Zahl (int, long) |
| `%f` | Gleitkommazahl (double, float) |
| `%n` | Zeilenumbruch (plattformunabhängig) |

### Text von der Konsole einlesen (Scanner)

Für die Eingabe über die Konsole verwendest du die Klasse `Scanner`:

```java
import java.util.Scanner;

public class EingabeDemo {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("Bitte Namen eingeben: ");
        String name = scanner.nextLine();

        System.out.print("Bitte Alter eingeben: ");
        int alter = scanner.nextInt();

        System.out.println("Hallo " + name + "! Du bist " + alter + " Jahre alt.");

        scanner.close();  // Ressource freigeben
    }
}
```

**Wichtige Scanner-Methoden:**

| Methode | Beschreibung |
|---------|-------------|
| `nextLine()` | Liest eine ganze Zeile als String |
| `next()` | Liest ein Wort (bis Leerzeichen) |
| `nextInt()` | Liest eine ganze Zahl |
| `nextDouble()` | Liest eine Gleitkommazahl |
| `hasNextInt()` | Prüft, ob eine ganze Zahl eingegeben wurde |
| `close()` | Scanner schließen (Ressource freigeben) |

**Achtung:** Nach `nextInt()` oder `nextDouble()` bleibt ein Zeilenumbruch im Puffer. Wenn du danach `nextLine()` aufrufst, bekommst du eine leere Zeile. Füge in dem Fall ein zusätzliches `nextLine()` ein:

```java
System.out.print("Alter: ");
int alter = scanner.nextInt();
scanner.nextLine();                    // Zeilenumbruch konsumieren
System.out.print("Diagnose: ");
String diagnose = scanner.nextLine();  // funktioniert jetzt korrekt
```

Vergleich mit Python:

```python
# Python
name = input("Bitte Namen eingeben: ")
alter = int(input("Bitte Alter eingeben: "))
print(f"Hallo {name}! Du bist {alter} Jahre alt.")
```
