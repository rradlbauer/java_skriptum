# 03.0 Hello World Programm ohne Entwicklungsumgebung

Bevor wir IntelliJ verwenden, erstellen wir ein Java-Programm mit einem einfachen Texteditor und der Kommandozeile. So verstehst du, was im Hintergrund passiert.

## Das erste Java-Programm

Erstelle eine Datei namens `HelloWorld.java` mit folgendem Inhalt:

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hallo, Welt!");
    }
}
```

## JDK-Tools: javac und java

Das JDK stellt zwei wichtige Kommandozeilen-Werkzeuge bereit:

### 1. Compiler: `javac`

Der Compiler übersetzt die `.java`-Datei in Byte Code:

```
javac HelloWorld.java
```

Nach erfolgreicher Kompilierung findest du eine neue Datei: `HelloWorld.class`. Das ist der Byte Code.

```
HelloWorld.java  →  (javac)  →  HelloWorld.class
```

### 2. Ausführen: `java`

Die Laufzeitumgebung führt den Byte Code aus:

```
java HelloWorld
```

Ausgabe:
```
Hallo, Welt!
```

**Wichtig:** Beim Ausführen mit `java` wird **ohne** `.class`-Endung angegeben.

## Die main-Methode im Detail

Jedes ausführbare Java-Programm benötigt eine `main`-Methode mit genau dieser Signatur:

```java
public static void main(String[] args)
```

| Teil | Bedeutung |
|------|-----------|
| `public` | Die Methode ist von außen zugänglich |
| `static` | Die Methode gehört zur Klasse, nicht zu einer Instanz |
| `void` | Die Methode gibt keinen Wert zurück |
| `main` | Name der Methode (vom JVM vorgegeben) |
| `String[] args` | Kommandozeilen-Argumente (wie `sys.argv` in Python) |

## Klassenname = Dateiname

In Java muss der **Dateiname exakt dem Klassennamen** entsprechen:

| Klasse | Dateiname |
|--------|-----------|
| `public class HelloWorld` | `HelloWorld.java` |
| `public class Patient` | `Patient.java` |

Ein Unterschied zu Python: In Python kann eine Datei `mein_programm.py` mehrere Klassen enthalten, und der Dateiname ist frei wählbar.

## Kompilierungsfehler kennenlernen

Ändere bewusst etwas am Code, um typische Fehlermeldungen zu sehen:

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hallo, Welt!")
        // Fehler: fehlendes Semikolon!
    }
}
```

Beim Kompilieren:

```
HelloWorld.java:3: error: ';' expected
    System.out.println("Hallo, Welt!")
                                       ^
1 error
```

Diese Fehler werden zur **Compile-Zeit** erkannt – ein Vorteil gegenüber Python, wo solche Fehler erst zur Laufzeit auftreten.

## Zusammenfassung

### Wichtige Begriffe
- `javac` (Java-Compiler)
- `java` (Java-Laufzeitumgebung)
- `main`-Methode
- Byte Code (`.class`)
- Compile-Zeit / Laufzeit
- Klassenname = Dateiname

### Das sollst du können
- Ein Java-Programm mit Texteditor schreiben
- Mit `javac` kompilieren
- Mit `java` ausführen
- Typische Compiler-Fehler erkennen und beheben
