# 07.0 Vererbung und Polymorphie

Vererbung kennst du bereits aus Python. In Java wird sie mit `extends` umgesetzt und ist strenger typisiert.

## Vererbung in Java

```java
public class Untersuchung {
    protected String patientName;
    protected String datum;

    public Untersuchung(String patientName, String datum) {
        this.patientName = patientName;
        this.datum = datum;
    }

    public void durchfuehren() {
        System.out.println("Untersuchung für " + patientName + " am " + datum);
    }
}
```

```java
public class Blutuntersuchung extends Untersuchung {
    private String blutwert;
    private double messwert;

    public Blutuntersuchung(String patientName, String datum, String blutwert, double messwert) {
        super(patientName, datum);  // Aufruf des Eltern-Konstruktors
        this.blutwert = blutwert;
        this.messwert = messwert;
    }
}
```

### super vs. super() 

- `super()` ruft den Konstruktor der Elternklasse auf **muss die erste Anweisung im Konstruktor sein**
- `super.methodenname()` ruft eine überschriebene Methode der Elternklasse auf

Vergleich mit Python:

| Python | Java |
|--------|------|
| `super().__init__(name)` | `super(name)` |
| `super().methode()` | `super.methode()` |

## Methoden überschreiben (@Override)

Eine Unterklasse kann Methoden der Oberklasse **überschreiben** (override):

```java
public class Blutuntersuchung extends Untersuchung {
    // ...

    @Override
    public void durchfuehren() {
        System.out.println("Blutuntersuchung für " + patientName);
        System.out.println("  Wert: " + blutwert + " = " + messwert);
    }
}
```

Die `@Override`-Annotation ist optional, aber empfohlen:
- Der Compiler prüft, ob die Methode tatsächlich eine Methode der Oberklasse überschreibt
- Schützt vor Tippfehlern (z. B. `durchfuehren()` vs. `durchführen()`)

Wichtig: In Python werden Methoden immer dynamisch überschrieben. In Java muss die Methodensignatur **exakt** übereinstimmen (Name + Parameter + Rückgabetyp).

## Polymorphie

Polymorphie bedeutet, dass eine Referenz vom Typ der Oberklasse auf ein Objekt der Unterklasse zeigen kann:

```java
Untersuchung u1 = new Untersuchung("Anna", "2024-01-15");
Untersuchung u2 = new Blutuntersuchung("Ben", "2024-01-15", "Blutzucker", 95.0);

u1.durchfuehren();  // Ruft Untersuchung.durchfuehren() auf
u2.durchfuehren();  // Ruft Blutuntersuchung.durchfuehren() auf (obwohl Typ Untersuchung!)
```

Das ist das Kernprinzip der Polymorphie: **Welche Methode aufgerufen wird, entscheidet der Objekttyp zur Laufzeit, nicht der Referenztyp zur Compile-Zeit.**

### Vergleich Python vs. Java

```python
# Python – Duck Typing
def verarbeite(untersuchung):
    untersuchung.durchfuehren()  # Hauptsache, die Methode existiert
```

```java
// Java – Polymorphie mit Vererbung
public void verarbeite(Untersuchung u) {
    u.durchfuehren();  // u MUSS vom Typ Untersuchung sein (oder Unterklasse)
}
```

Java ist hier strenger: Der Typ muss zur Compile-Zeit bekannt sein. Python akzeptiert jedes Objekt, das die Methode hat (Duck Typing).

## Protected

`protected` erlaubt den Zugriff in:
- Derselben Klasse
- Demselben Package
- Allen Unterklassen (auch in anderen Packages)

Das ist ideal für Attribute, die in Unterklassen verwendet werden sollen.

## Zusammenfassung

### Wichtige Begriffe
- `extends`
- `super()`
- Vererbung (inheritance)
- Überschreiben (override)
- `@Override`-Annotation
- Polymorphie
- `protected`

### Das sollst du können
- Eine Klassenhierarchie mit `extends` aufbauen
- Methoden mit `@Override` überschreiben
- Den Unterschied zwischen Referenztyp und Objekttyp erklären
- Polymorphie anwenden
