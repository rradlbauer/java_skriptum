# 09.0 Generics (Typparameter)

In Python sind Listen und Dictionaries typschwach – eine Liste kann alles Mögliche enthalten. In Java gibt es **Generics** (Typparameter), um zur Compile-Zeit festzulegen, welche Typen in einer Klasse oder Methode verwendet werden dürfen.

## Motivation

**Python** erlaubt alles zur Laufzeit:

```python
python_liste = ["Anna", 42, 3.14, True]  # alles erlaubt
```

**Java** verhindert solche Mischtypen bereits beim Kompilieren:

```java
List<String> namen = new ArrayList<>();   // nur Strings
// namen.add(42);                         // Compiler-Fehler!
```

Vorteile:
- **Typsicherheit:** Falsche Typen werden bereits beim Kompilieren erkannt
- **Keine Casts:** Du musst Elemente nicht manuell von `Object` nach `String` casten
- **Klarer Code:** Der Leser sieht sofort, welche Typen verwendet werden

## Generic-Klassen

Du kannst eigene Klassen mit Typparametern definieren:

```java
public class Box<T> {
    private T inhalt;

    public void setInhalt(T inhalt) {
        this.inhalt = inhalt;
    }

    public T getInhalt() {
        return inhalt;
    }
}
```

Verwendung:

```java
Box<String> stringBox = new Box<>();
stringBox.setInhalt("Hallo");
String wert = stringBox.getInhalt();  // Kein Cast nötig!

Box<Integer> intBox = new Box<>();
intBox.setInhalt(42);
int zahl = intBox.getInhalt();        // Autoboxing: Integer → int
```

Der Typparameter `T` ist ein Platzhalter für einen konkreten Typ. Übliche Namen:
- `T` – Type (allgemein)
- `E` – Element (z. B. in Collections)
- `K`, `V` – Key, Value (z. B. in Maps)
- `N` – Number
- `R` – Return type

## Mehrere Typparameter

Eine Klasse kann mehrere Typparameter haben:

```java
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }
}
```

```java
Pair<String, Integer> patient = new Pair<>("Anna", 25);
String name = patient.getKey();
int alter = patient.getValue();
```

## Generic-Methoden

Nicht nur Klassen, auch einzelne Methoden können Typparameter haben:

```java
public class Utils {
    public static <T> void printList(List<T> list) {
        for (T element : list) {
            System.out.println(element);
        }
    }
}
```

Der Typparameter `T` wird vor dem Rückgabetyp deklariert:

```java
public static <T> T getLetztes(List<T> list) {
    return list.get(list.size() - 1);
}
```

## Bounded Type Parameters (Einschränkungen)

Du kannst Typparameter auf bestimmte Typen einschränken:

```java
// T muss Number oder eine Unterklasse sein
public static <T extends Number> double summe(List<T> zahlen) {
    double summe = 0;
    for (T n : zahlen) {
        summe += n.doubleValue();
    }
    return summe;
}
```

```java
List<Integer> ints = List.of(1, 2, 3);
List<String> strings = List.of("A", "B");

double s = summe(ints);     // OK: Integer extends Number
// double s2 = summe(strings); // Compiler-Fehler: String extends nicht Number
```

### Bounds mit Interfaces

```java
public static <T extends Comparable<T>> T max(T a, T b) {
    return a.compareTo(b) > 0 ? a : b;
}
```

Nur Typen, die `Comparable` implementieren, können verwendet werden:

```java
System.out.println(max(5, 10));        // 10 (Integer implements Comparable)
System.out.println(max("Anna", "Ben")); // "Ben"
```

## Wildcards `?`

Manchmal weißt du den genauen Typ nicht oder er ist egal:

### Unbounded Wildcard `?`

```java
public static void printAll(List<?> list) {
    for (Object o : list) {
        System.out.println(o);
    }
}
```

`List<?>` bedeutet "Liste von irgendwas". Du kannst sie lesen, aber keine Elemente hinzufügen (außer `null`).

### Upper Bounded Wildcard `? extends T`

`? extends T` akzeptiert `T` oder jeden Subtyp von `T`:

```java
public static double summe(List<? extends Number> zahlen) {
    double sum = 0;
    for (Number n : zahlen) {
        sum += n.doubleValue();
    }
    return sum;
}
```

```java
List<Integer> ints = List.of(1, 2, 3);
List<Double> doubles = List.of(1.5, 2.5);

summe(ints);     // OK
summe(doubles);  // OK
```

### Lower Bounded Wildcard `? super T`

`? super T` akzeptiert `T` oder jeden Supertyp von `T`:

```java
public static void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}
```

```java
List<Number> numbers = new ArrayList<>();
addNumbers(numbers);  // OK: Number ist super von Integer
```

**Faustregel (PECS):** Producer `extends`, Consumer `super` – wenn du Werte aus einer Struktur liest (producer), verwende `? extends T`; wenn du Werte einfügst (consumer), verwende `? super T`.

## Diamond-Operator `<>`

Seit Java 7 kannst du den Typ auf der rechten Seite weglassen – der Compiler leitet ihn aus der Deklaration ab:

```java
List<String> namen = new ArrayList<>();   // <> statt <String>
Map<String, Integer> map = new HashMap<>();

Box<Patient> box = new Box<>();           // Typ wird von links übernommen
```

Ohne Diamond-Operator müsstest du schreiben:

```java
List<String> namen = new ArrayList<String>();  // Typ-Wiederholung (vor Java 7)
```

## Type Erasure (Typ-Löschung)

Generics gibt es nur zur Compile-Zeit. Im Byte Code werden die Typparameter gelöscht (durch `Object` oder den Bound ersetzt):

```java
// Java-Code:
List<String> namen = new ArrayList<>();

// Nach Type Erasure (Byte Code):
List namen = new ArrayList();  // Kein Typ-Parameter mehr
```

Das hat Konsequenzen:
- `instanceof` mit Generics ist nicht möglich: `list instanceof List<String>` → Compiler-Fehler
- Du kannst keine Arrays mit Generics erstellen: `new T[10]` → Compiler-Fehler
- Zur Laufzeit ist der Typ nicht mehr bekannt

## Vergleich Python vs. Java

| Python | Java |
|--------|------|
| `def box(inhalt):` | `public class Box<T> { ... }` |
| `box = Box("Anna")` | `Box<String> box = new Box<>()` |
| `box.inhalt = 42` (erlaubt) | `box.setInhalt(42)` (Compiler-Fehler) |
| Typprüfung zur Laufzeit | Typprüfung zur Compile-Zeit |
| Keine Generics nötig (duck typing) | Generics für Typsicherheit |

## Zusammenfassung

### Wichtige Begriffe
- Generic-Klasse / Generic-Methode
- Typparameter `<T>`
- Diamond-Operator `<>`
- Bounded Type Parameter `<T extends ...>`
- Wildcards (`?`, `? extends`, `? super`)
- Type Erasure
- PECS (Producer Extends, Consumer Super)

### Das sollst du können
- Eigene Generic-Klassen und -Methoden definieren
- Typparameter mit Bounds einschränken
- Wildcards (`? extends`, `? super`) korrekt verwenden
- Den Diamond-Operator anwenden
- Type Erasure und ihre Konsequenzen erklären
- Den Vorteil von Generics gegenüber Python's dynamischer Typisierung erklären
