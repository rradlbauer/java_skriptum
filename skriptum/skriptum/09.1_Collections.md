# 09.1 Java Collections – Grundlagen

In Python kennst du Listen, Dictionaries und Sets als eingebaute Datentypen. In Java gibt es das **Collections Framework** – eine einheitliche Architektur zum Speichern und Verwalten von Objektgruppen.

> Generics (Typparameter) werden in diesem Kapitel verwendet. Details dazu findest du in **[Kapitel 09.0](09.0_Generics.md)**.

## Wiederholung Python

```python
# Python: flexible, eingebaute Datentypen
liste = [1, 2, 3]
menge = {1, 2, 3}
dict = {"name": "Anna", "alter": 25}
```

In Java werden Sammlungen über eine Hierarchie von Interfaces bereitgestellt. Du musst dich für einen konkreten Typ entscheiden.

## Vererbungshierarchie der Collections

```
Iterable<T> (Interface)
    └── Collection<T> (Interface)
            ├── List<T> (Interface)
            ├── Set<T> (Interface)
            └── Queue<T> (Interface)

Map<K, V> (Interface, eigenständig)
```

- **`Iterable<T>`** – die Basis: ermöglicht die for-each-Schleife
- **`Collection<T>`** – die Sammlung: Größe, Hinzufügen, Entfernen, Prüfen
- **`List<T>`** – geordnete Liste mit Indexzugriff (wie Python-Liste)
- **`Set<T>`** – Menge ohne Duplikate (wie Python-Set)
- **`Map<K, V>`** – Schlüssel-Wert-Paare (wie Python-Dict)

## Iterable<T>

`Iterable<T>` ist das grundlegende Interface. Alles, was `Iterable` implementiert, kann in einer for-each-Schleife verwendet werden:

```java
import java.util.List;
import java.util.ArrayList;

List<String> namen = new ArrayList<>();
namen.add("Anna");
namen.add("Ben");
namen.add("Clara");

for (String name : namen) {
    System.out.println(name);
}
```

Das Interface `Iterable<T>` hat nur eine Methode:

```java
public interface Iterable<T> {
    Iterator<T> iterator();
}
```

Der `Iterator` erlaubt dir, manuell durch die Elemente zu gehen:

```java
Iterator<String> it = namen.iterator();
while (it.hasNext()) {
    String name = it.next();
    System.out.println(name);
}
```

## Collection<T>

`Collection<T>` erweitert `Iterable<T>` und definiert grundlegende Methoden, die in **allen** Collection-Typen verfügbar sind:

| Methode | Beschreibung |
|---------|-------------|
| `boolean add(T e)` | Element hinzufügen |
| `boolean remove(Object o)` | Element entfernen |
| `boolean contains(Object o)` | Prüfen, ob Element enthalten ist |
| `int size()` | Anzahl der Elemente |
| `boolean isEmpty()` | Ist die Sammlung leer? |
| `void clear()` | Alle Elemente entfernen |
| `Object[] toArray()` | In Array umwandeln |

## Autoboxing

Java wandelt primitive Typen automatisch in ihre Wrapper-Klassen um:

```java
List<Integer> zahlen = new ArrayList<>();
zahlen.add(42);              // Autoboxing: int → Integer
int wert = zahlen.get(0);    // Unboxing: Integer → int
```

Das funktioniert für alle primitiven Typen (`int` ↔ `Integer`, `double` ↔ `Double`, `boolean` ↔ `Boolean`, ...).
