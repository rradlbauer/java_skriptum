# 07.1 Abstrakte Klassen und Interfaces

Abstrakte Klassen und Interfaces sind zentrale Werkzeuge, um **Schnittstellen von Implementierungen zu trennen**. In Python gibt es dafür das `abc`-Modul.

## Abstrakte Klassen

Eine **abstrakte Klasse** kann nicht direkt instanziiert werden. Sie dient als Vorlage für Unterklassen:

```java
public abstract class Untersuchung {
    protected String patientName;
    protected String datum;

    public Untersuchung(String patientName, String datum) {
        this.patientName = patientName;
        this.datum = datum;
    }

    // Abstrakte Methode: keine Implementierung, muss von Unterklassen überschrieben werden
    public abstract void durchfuehren();

    // Konkrete Methode: wird vererbt
    public String getPatientName() {
        return patientName;
    }
}
```

```java
public class Blutuntersuchung extends Untersuchung {
    private String blutwert;
    private double messwert;

    public Blutuntersuchung(String patientName, String datum, String blutwert, double messwert) {
        super(patientName, datum);
        this.blutwert = blutwert;
        this.messwert = messwert;
    }

    @Override
    public void durchfuehren() {   // Muss implementiert werden!
        System.out.println(blutwert + ": " + messwert);
    }
}
```

```java
// Untersuchung u = new Untersuchung(...);  // Compiler-Fehler! abstrakte Klasse
Untersuchung u = new Blutuntersuchung("Anna", "2024-01-15", "HbA1c", 5.7);
u.durchfuehren();
```

## Interfaces

Ein **Interface** definiert einen Vertrag (eine Schnittstelle), den eine Klasse erfüllen muss. Es enthält nur Methodensignaturen (keine Implementierung, keine Attribute).

```java
public interface Behandelbar {
    void behandeln();               // automatisch public abstract
    String getDiagnose();           // automatisch public abstract
}
```

```java
public class Patient implements Behandelbar {
    private String name;
    private String diagnose;

    public Patient(String name, String diagnose) {
        this.name = name;
        this.diagnose = diagnose;
    }

    @Override
    public void behandeln() {
        System.out.println(name + " wird behandelt für: " + diagnose);
    }

    @Override
    public String getDiagnose() {
        return diagnose;
    }
}
```

### Mehrere Interfaces implementieren

Anders als bei Vererbung (nur eine Oberklasse) kann eine Klasse **mehrere Interfaces implementieren**:

```java
public interface Impfbar {
    void impfen(String impfstoff);
}

public interface Untersuchbar {
    void untersuchen(String art);
}

public class Patient implements Behandelbar, Impfbar, Untersuchbar {
    // Muss ALLE Methoden aller Interfaces implementieren
    @Override public void behandeln() { /* ... */ }
    @Override public String getDiagnose() { return ""; }
    @Override public void impfen(String impfstoff) { /* ... */ }
    @Override public void untersuchen(String art) { /* ... */ }
}
```

### Default-Methoden in Interfaces (Java 8+)

Seit Java 8 können Interfaces auch **default-Methoden** mit einer Standard-Implementierung enthalten:

```java
public interface Behandelbar {
    void behandeln();

    default void dokumentieren() {
        System.out.println("Behandlung dokumentiert.");
    }
}
```

## Abstrakte Klasse vs. Interface

| Kriterium | Abstrakte Klasse | Interface |
|-----------|:---:|:---:|
| Kann Attribute haben | ✓ | ✗ |
| Kann Konstruktoren haben | ✓ | ✗ |
| Kann normale Methoden haben | ✓ | nur default-Methoden |
| Kann abstrakte Methoden haben | ✓ | ✓ |
| Mehrfachvererbung möglich | ✗ (nur eine O.K.) | ✓ (mehrere Interfaces) |
| Verwendung | `extends` | `implements` |

**Faustregel:**
- Verwende eine **abstrakte Klasse**, wenn die Klassen eine gemeinsame Basis mit Code teilen
- Verwende ein **Interface**, wenn du einen Vertrag definieren willst, den viele unterschiedliche Klassen erfüllen können

## Trennung von Schnittstelle und Implementierung

Interfaces ermöglichen die saubere Trennung von "was" und "wie":

```java
// Schnittstelle (was)
public interface DiagnoseTool {
    String analysieren(String daten);
}

// Implementierung 1 (wie)
public class BlutAnalyse implements DiagnoseTool {
    @Override
    public String analysieren(String daten) {
        return "Blutanalyse: " + daten;
    }
}

// Implementierung 2 (wie)
public class MRT implements DiagnoseTool {
    @Override
    public String analysieren(String daten) {
        return "MRT-Befund: " + daten;
    }
}

// Verwendung (Programmieren gegen die Schnittstelle)
public class Arzt {
    public void diagnostizieren(DiagnoseTool tool, String daten) {
        String ergebnis = tool.analysieren(daten);
        System.out.println(ergebnis);
    }
}
```

## Vergleich mit Python

```python
# Python – abc-Modul
from abc import ABC, abstractmethod

class Untersuchung(ABC):
    @abstractmethod
    def durchfuehren(self):
        pass

# Python hat kein echtes Interface-Konzept (es wird ABC dafür verwendet)
```

In Java wird die Trennung von Schnittstelle und Implementierung durch die Sprachsyntax erzwungen.

## Zusammenfassung

### Wichtige Begriffe
- `abstract class` / `abstract method`
- `interface` / `implements`
- `default`-Methode
- Programmieren gegen die Schnittstelle
- Vertrag (contract)

### Das sollst du können
- Abstrakte Klassen definieren und verwenden
- Interfaces definieren und implementieren
- Mehrere Interfaces kombinieren
- Den Unterschied zwischen abstrakter Klasse und Interface erklären
- Gegen Schnittstellen statt gegen Implementierungen programmieren
