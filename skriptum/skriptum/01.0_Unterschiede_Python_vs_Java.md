# 01.0 Wesentliche Unterschiede: Python vs. Java

Python und Java sind zwei der populärsten Programmiersprachen, unterscheiden sich jedoch grundlegend in ihrer Arbeitsweise.

## Skriptsprache vs. Compilersprache

**Python** ist eine interpretierte Sprache:
- Python-Code wird zur Laufzeit Zeile für Zeile von einem Interpreter ausgeführt
- Code kann sofort getestet werden (z. B. in der interaktiven Konsole)
- Fehler werden erst zur Laufzeit erkannt

**Java** ist eine compilierte Sprache:
- Java-Code muss vor der Ausführung übersetzt (kompiliert) werden
- Viele Fehler werden bereits beim Kompilieren erkannt (Typfehler, Syntaxfehler)
- Der Kompilierungsprozess erzeugt **Byte Code**

## Was macht ein Compiler?

Ein Compiler übersetzt den für Menschen lesbaren Quellcode in eine für die Maschine ausführbare Form.

```java
// Quellcode (HelloWorld.java)
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hallo Welt!");
    }
}
```

```
 ↓ javac (Java-Compiler)
```

```
HelloWorld.class (Byte Code - für Menschen nicht lesbar)
```

## Source Code → Byte Code

| Phase | Beschreibung |
|-------|-------------|
| **Source Code** | `.java`-Dateien mit menschenlesbarem Code |
| **Compiler** | `javac` übersetzt `.java` → `.class` |
| **Byte Code** | `.class`-Dateien enthalten plattformunabhängigen Byte Code |

## Die virtuelle Maschine (JVM)

Der Byte Code wird nicht direkt vom Betriebssystem ausgeführt, sondern von der **Java Virtual Machine (JVM)**:

```text
Quellcode (.java) → Compiler (javac) → Byte Code (.class) → JVM → Betriebssystem
```

Die JVM ist ein Programm, das den Byte Code interpretiert und zur Laufzeit weiter optimiert (Just-In-Time-Kompilierung). Dadurch ist Java plattformunabhängig: Derselbe Byte Code läuft auf Windows, Linux oder macOS, sofern eine JVM installiert ist.

## Weitere wichtige Unterschiede

| Aspekt | Python | Java |
|--------|--------|------|
| Typisierung | dynamisch (Typ wird zur Laufzeit bestimmt) | statisch (Typ wird zur Compile-Zeit festgelegt) |
| Syntax | einrückungsbasiert, keine Semikolons | blockstrukturiert mit `{}`, Semikolons nötig |
| Dateiname | beliebig (`.py`) | muss dem Klassennamen entsprechen (`.java`) |
| Ausführung | `python datei.py` | erst `javac Datei.java`, dann `java Datei` |
