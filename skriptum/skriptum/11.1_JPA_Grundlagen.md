# 11.1 JPA – Grundlagen (EntityManager, ORM)

**JPA** (Java Persistence API) ist der Standard für **Objektrelationales Mapping (ORM)** in Java. Statt SQL-Befehle manuell zu schreiben (wie bei JDBC), arbeitest du mit Java-Objekten – JPA übersetzt die Operationen automatisch in SQL.

## Überblick

### Was ist ORM?

ORM bildet Java-Klassen auf Datenbanktabellen ab:

```
Java-Klasse Patient  →  Tabelle patient
Java-Objekt p1       →  Zeile in patient
Attribut name        →  Spalte name
```

Vorteile gegenüber JDBC:
- Kein manuelles SQL (JPA generiert es)
- Kein `ResultSet`-Mapping von Hand
- Automatische Transaktionsverwaltung
- Wechsel der Datenbank oft ohne Code-Änderungen

### JPA-Implementierungen

JPA ist nur eine Spezifikation. Du benötigst eine Implementierung:

- **Hibernate** (am weitesten verbreitet)
- EclipseLink
- OpenJPA

In diesem Kapitel verwenden wir Hibernate.

## Konfiguration mit persistence.xml

JPA wird über die Datei `META-INF/persistence.xml` konfiguriert (im Ressourcen-Ordner `src/main/resources/META-INF/`):

```xml
<persistence xmlns="https://jakarta.ee/xml/ns/persistence"
             version="3.0">
    <persistence-unit name="medizinPU">
        <provider>org.hibernate.jpa.HibernatePersistenceProvider</provider>
        <class>at.htl.medizin.model.Patient</class>

        <properties>
            <property name="jakarta.persistence.jdbc.url"
                      value="jdbc:mysql://localhost:3306/medizin"/>
            <property name="jakarta.persistence.jdbc.user" value="root"/>
            <property name="jakarta.persistence.jdbc.password" value="passwort"/>
            <property name="jakarta.persistence.jdbc.driver"
                      value="com.mysql.cj.jdbc.Driver"/>

            <!-- Hibernate erstellt Tabellen automatisch -->
            <property name="hibernate.hbm2ddl.auto" value="update"/>
            <property name="hibernate.show_sql" value="true"/>
        </properties>
    </persistence-unit>
</persistence>
```

Wichtige Properties:

| Property | Bedeutung |
|----------|-----------|
| `hibernate.hbm2ddl.auto` | `update`: Tabellen automatisch erstellen/aktualisieren; `create-drop`: nur zur Laufzeit; `validate`: nur prüfen |
| `hibernate.show_sql` | Zeigt generierte SQL-Befehle in der Konsole |
| `hibernate.format_sql` | Formatiert die SQL-Ausgabe lesbar |

## ORM-Annotations

Eine JPA-Entity wird mit Annotationen versehen:

```java
package at.htl.medizin.model;

import jakarta.persistence.*;

@Entity
@Table(name = "patient")
public class Patient {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private int id;

    @Column(name = "name", nullable = false, length = 100)
    private String name;

    @Column(name = "alter")
    private int alter;

    @Column(name = "diagnose")
    private String diagnose;

    // Konstruktor (standard+parametrisiert)
    public Patient() {}
    public Patient(String name, int alter, String diagnose) {
        this.name = name;
        this.alter = alter;
        this.diagnose = diagnose;
    }

    // Getter / Setter
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }
    // ... weitere Getter/Setter
}
```

Wichtige Annotationen:

| Annotation | Beschreibung |
|-----------|-------------|
| `@Entity` | Markiert die Klasse als JPA-Entity (wird auf Tabelle abgebildet) |
| `@Table(name)` | Optional: Name der Datenbanktabelle (sonst Klassenname) |
| `@Id` | Primärschlüssel |
| `@GeneratedValue(strategy = IDENTITY)` | Auto-increment (wie MySQL AUTO_INCREMENT) |
| `@Column(name, nullable, length)` | Optional: Spalten-Details |

### GenerationType.IDENTITY

Für MySQL mit `AUTO_INCREMENT` verwendest du `GenerationType.IDENTITY`. Die ID wird erst beim Speichern von der Datenbank vergeben.

## EntityManager

Der `EntityManager` ist die zentrale Schnittstelle für alle Datenbankoperationen. Du erhältst ihn über eine `EntityManagerFactory`:

```java
import jakarta.persistence.*;

EntityManagerFactory emf = Persistence.createEntityManagerFactory("medizinPU");
EntityManager em = emf.createEntityManager();
```

Nach der Verwendung schließt du beide:

```java
em.close();
emf.close();
```

### Transaktionen

Jede Änderung an der Datenbank muss in einer Transaktion erfolgen:

```java
em.getTransaction().begin();
// ... Änderungen ...
em.getTransaction().commit();
```

Bei Fehlern rollst du zurück:

```java
em.getTransaction().begin();
try {
    // ... Änderungen ...
    em.getTransaction().commit();
} catch (Exception e) {
    em.getTransaction().rollback();
    throw e;
}
```

## Objekte persistieren (INSERT)

```java
Patient p = new Patient("Anna Müller", 34, "Hypertonie");

em.getTransaction().begin();
em.persist(p);                       // INSERT
em.getTransaction().commit();

System.out.println("ID: " + p.getId());  // generierte ID ist jetzt gesetzt
```

Nach `persist()` ist das Objekt **managed** (vom Persistence Context verwaltet).

## Objekt abrufen (SELECT)

```java
Patient p = em.find(Patient.class, 1);   // SELECT WHERE id = 1
System.out.println(p.getName());
```

`find()` gibt `null` zurück, wenn kein Datensatz mit der ID existiert.

## Liste von Objekten abrufen (JPQL)

**JPQL** (Java Persistence Query Language) ähnelt SQL, arbeitet aber mit Klassen- und Attributnamen statt Tabellen- und Spaltennamen:

```java
TypedQuery<Patient> query = em.createQuery(
    "SELECT p FROM Patient p WHERE p.diagnose = :diag", Patient.class);
query.setParameter("diag", "Hypertonie");

List<Patient> liste = query.getResultList();
```

Vergleich JPQL vs. SQL:

```java
// JPQL:  SELECT p FROM Patient p WHERE p.alter > :min
// SQL:   SELECT * FROM patient WHERE alter > ?
```

Weitere JPQL-Beispiele:

```java
// Alle Patienten
TypedQuery<Patient> alle = em.createQuery("SELECT p FROM Patient p", Patient.class);

// Nur Namen abrufen
TypedQuery<String> namen = em.createQuery(
    "SELECT p.name FROM Patient p WHERE p.diagnose = :diag", String.class);

// Mit count
TypedQuery<Long> anzahl = em.createQuery(
    "SELECT COUNT(p) FROM Patient p", Long.class);
```

## Objekt aktualisieren (UPDATE)

Ein **managed**‑Objekt kannst du einfach ändern – JPA speichert die Änderungen automatisch beim `commit()`:

```java
em.getTransaction().begin();

Patient p = em.find(Patient.class, 1);   // managed
p.setDiagnose("Diabetes");                // Änderung wird automatisch erkannt

em.getTransaction().commit();             // UPDATE wird generiert
```

Für getrennte (detached) Objekte verwendest du `merge()`:

```java
Patient p = new Patient();
p.setId(1);
p.setName("Anna Müller");
p.setAlter(34);
p.setDiagnose("Diabetes");

em.getTransaction().begin();
em.merge(p);                              // UPDATE oder INSERT
em.getTransaction().commit();
```

## Objekt löschen (DELETE)

```java
em.getTransaction().begin();

Patient p = em.find(Patient.class, 1);   // managed
em.remove(p);                              // DELETE

em.getTransaction().commit();
```

## Persistence Context

Der **Persistence Context** ist ein Zwischenspeicher (first-level cache), den der `EntityManager` verwaltet:

```
em.find(Patient.class, 1)   → Datenbank → Persistence Context → Objekt
em.find(Patient.class, 1)   → Persistence Context (kein DB-Zugriff!)
```

Wichtige Zustände einer Entity:

| Zustand | Beschreibung |
|---------|-------------|
| **new** | Noch nicht in der DB, nicht im Persistence Context |
| **managed** | Im Persistence Context, mit DB synchronisiert |
| **detached** | War im Persistence Context, wurde getrennt (z. B. nach `em.close()`) |
| **removed** | Zum Löschen vorgemerkt (beim commit gelöscht) |

## Zusammenfassung

### Wichtige Begriffe
- JPA (Java Persistence API)
- ORM (Objektrelationales Mapping)
- `@Entity` / `@Id` / `@GeneratedValue`
- `persistence.xml`
- `EntityManagerFactory` / `EntityManager`
- `persist()` / `find()` / `merge()` / `remove()`
- JPQL (`createQuery`, `TypedQuery`)
- Transaktion (`begin`, `commit`, `rollback`)
- Persistence Context (managed, detached)

### Das sollst du können
- Eine `persistence.xml` für MySQL konfigurieren
- Eine Entity-Klasse mit JPA-Annotationen erstellen
- Objekte mit `persist()` in der Datenbank speichern
- Objekte mit `find()` und JPQL abrufen
- Objekte mit `merge()` aktualisieren
- Objekte mit `remove()` löschen
- Transaktionen korrekt verwalten
- Den Persistence Context und seine Zustände erklären
