# 05.0 Wiederholung grundlegender OOP-Konzepte

Du kennst objektorientierte Programmierung bereits aus Python. In Java wird OOP noch strenger umgesetzt – Java ist eine **rein objektorientierte Sprache**.

## Klassen und Instanzen

Eine Klasse in Java ähnelt einer Klasse in Python. Der grundlegende Aufbau unterscheidet sich jedoch in der Syntax.

**Python:**
```python
class Patient:
    def __init__(self, name):
        self.name = name
```

**Java:**
```java
public class Patient {
    String name;

    public Patient(String name) {
        this.name = name;
    }
}
```

## Klasse als Datentyp

In Java ist eine Klasse immer auch ein **Datentyp**. Du kannst Variable vom Typ einer Klasse deklarieren:

```java
Patient p1;              // Variable vom Typ Patient
p1 = new Patient("Anna"); // Objekt erzeugen mit new

Patient p2 = new Patient("Ben");  // Deklaration + Initialisierung
```

Der `new`-Operator erzeugt ein neues Objekt. Ohne `new` hast du nur eine Variable, die `null` enthält (vergleichbar mit `None` in Python).

## Attribute und Methoden

**Attribute** (Instanzvariablen) speichern den Zustand eines Objekts, **Methoden** definieren sein Verhalten.

```java
public class Patient {
    // Attribute
    String name;
    int alter;
    double koerpergroesse;

    // Methode
    void ausgabe() {
        System.out.println("Name: " + name + ", Alter: " + alter);
    }
}
```

Verwendung:

```java
Patient p = new Patient();
p.name = "Anna";
p.alter = 25;
p.ausgabe();
```

## Parameter und Rückgabewerte

Methoden können Parameter entgegennehmen und Werte zurückgeben:

```java
public double berechneBMI(double gewicht) {
    return gewicht / (koerpergroesse * koerpergroesse);
}
```

Wichtig: Der **Rückgabetyp** muss bei der Methodendeklaration angegeben werden:

- `void` – keine Rückgabe
- `int`, `double`, `boolean`, ... – primitiver Rückgabetyp
- `String`, `Patient`, ... – Klassen-Datentyp als Rückgabetyp

## Wichtiger Unterschied: this/self ist kein Parameter

Anders als in Python ist `this` in Java **kein Methodenparameter**. In Python musst du `self` immer als ersten Parameter angeben:

```python
# Python: self ist ein expliziter Parameter
def ausgabe(self):              # self als erster Parameter
    print(self.name)
```

In Java ist `this` **implizit** immer verfügbar – du schreibst es nicht in die Parameterliste:

```java
// Java: this ist kein Parameter, sondern implizit verfügbar
void ausgabe() {                  // kein this in den Klammern
    System.out.println(this.name);
}
```

Das ist nicht nur ein syntaktischer Unterschied, sondern eine grundlegende Designentscheidung: Java-Methoden haben eine feste Signatur ohne versteckten Parameter.

## Vergleich Python vs. Java

| Konzept | Python | Java |
|---------|--------|------|
| Objekt erzeugen | `Patient("Anna")` | `new Patient("Anna")` |
| Self/this | `self.name = name` (`self` ist Parameter) | `this.name = name` (`this` ist implizit, kein Parameter) |
| Methode definieren | `def methode(self):` | `void methode() { }` |
| Rückgabetyp | nicht angegeben | muss angegeben werden |
| Attribut-Zugriff | `objekt.attribut` | `objekt.attribut` |
| `__init__` vs. Konstruktor | `__init__(self)` | `public Klasse(Param) { }` |
