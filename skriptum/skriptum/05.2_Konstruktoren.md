# 05.2 Konstruktoren, Getter/Setter und this

## Konstruktoren

Ein **Konstruktor** initialisiert ein neues Objekt. In Python heißt er `__init__`, in Java hat er **denselben Namen wie die Klasse** und **keinen Rückgabetyp** (nicht einmal `void`).

```java
public class Patient {
    private String name;
    private int alter;

    // Konstruktor
    public Patient(String name, int alter) {
        this.name = name;
        this.alter = alter;
    }
}
```

Erzeugung eines Objekts:

```java
Patient p = new Patient("Anna", 25);
```

### Default-Konstruktor

Wenn du **keinen** Konstruktor definierst, erstellt Java automatisch einen parameterlosen Default-Konstruktor. Sobald du einen eigenen Konstruktor definierst, gibt es keinen Default-Konstruktor mehr.

```java
public class Patient {
    private String name;

    // KEIN eigener Konstruktor → Default-Konstruktor vorhanden
}

// Verwendung:
Patient p = new Patient();  // Default-Konstruktor
```

## this vs. self

`this` in Java entspricht `self` in Python. Es verweist auf das aktuelle Objekt.

```java
public class Patient {
    private String name;

    public Patient(String name) {
        this.name = name;  // this.name = Attribut, name = Parameter
    }
}
```

Unterschiede:

| Aspekt | Python | Java |
|--------|--------|------|
| Schlüsselwort | `self` | `this` |
| Muss angegeben werden? | Ja, als erster Parameter | Nein, es ist implizit |
| Muss verwendet werden? | Ja, bei Attributzugriff | Nur bei Namenskonflikten nötig |

In Java kannst du `this` weglassen, wenn es keine Verwechslung gibt:

```java
public void ausgabe() {
    System.out.println(name);  // this.name ist implizit
}
```

`this` funktioniert genauso wie `self` in Python – du kannst damit nicht nur auf Attribute, sondern auch auf Methoden zugreifen:

```java
public class Patient {
    private String name;

    public void setzeName(String name) {
        this.name = name;
    }

    public void ausgabe() {
        this.setzeName("Anna");   // Methodenaufruf mit this (optional)
        System.out.println(this.name);
    }
}
```

Wie bei Attributen ist `this` auch bei Methoden optional – solange es keine Verwechslung gibt. Der Aufruf `setzeName("Anna")` ist identisch zu `this.setzeName("Anna")`.

## Getter und Setter

Da Attribute `private` sind, braucht es öffentliche Zugriffsmethoden:

```java
public class Patient {
    private String name;
    private double koerpergroesse;  // in Meter

    // Konstruktor
    public Patient(String name, double koerpergroesse) {
        this.name = name;
        this.koerpergroesse = koerpergroesse;
    }

    // Getter
    public String getName() {
        return name;
    }

    // Getter
    public double getKoerpergroesse() {
        return koerpergroesse;
    }

    // Setter mit Validierung
    public void setKoerpergroesse(double koerpergroesse) {
        if (koerpergroesse > 0 && koerpergroesse < 3.0) {
            this.koerpergroesse = koerpergroesse;
        }
    }

    // Setter (einfach)
    public void setName(String name) {
        this.name = name;
    }
}
```

### Konventionen für Getter/Setter

| Attribut | Getter | Setter |
|----------|--------|--------|
| `name` | `getName()` | `setName(String name)` |
| `alter` | `getAlter()` | `setAlter(int alter)` |
| `koerperGroesse` | `getKoerperGroesse()` | `setKoerperGroesse(double kg)` |
| `boolean aktiv` | `isAktiv()` | `setAktiv(boolean aktiv)` |

Das Setzen von Gettern/Settern kann IntelliJ für dich übernehmen: Rechtsklick → **Generate → Getter and Setter** (oder `Alt + Einfg`).
