# 03.1 Packages und voll qualifizierte Klassennamen

In Python organisierst du Code mit **Modulen** (Dateien) und **Packages** (Ordnern). Java verwendet ein ähnliches, aber strengeres System.

## Was sind Packages?

Packages in Java sind vergleichbar mit Python-Packages – sie dienen der Strukturierung von Code und vermeiden Namenskonflikte.

```text
Python:   import csv
Java:     import java.util.Scanner;
```

## Packages und Ordnerstruktur

Ein Package entspricht einem Ordner auf der Festplatte:

```java
package at.htl.medizin;

public class Patient {
    // ...
}
```

Diese Datei muss im Ordner `at/htl/medizin/` (bzw. `at\htl\medizin\` unter Windows) liegen.

### Beispiel: Vollständige Projektstruktur

```
myproject/
├── src/
│   └── at/
│       └── htl/
│           └── medizin/
│               ├── Patient.java          (package at.htl.medizin)
│               └── Krankenhaus.java      (package at.htl.medizin)
└── bin/
```

### Kompilieren mit Packages

```bash
javac -d bin src/at/htl/medizin/Patient.java
javac -d bin src/at/htl/medizin/Krankenhaus.java
```

Der Parameter `-d bin` legt fest, dass die `.class`-Dateien im `bin`-Ordner abgelegt werden. Der Compiler erstellt automatisch die Package-Ordnerstruktur.

### Ausführen mit Packages

```bash
java -cp bin at.htl.medizin.Krankenhaus
```

Die Option `-cp` (classpath) gibt an, wo die `.class`-Dateien zu finden sind.

## Voll qualifizierte Klassennamen

Der **voll qualifizierte Klassenname** (FQN) setzt sich aus Package + Klassenname zusammen:

```text
at.htl.medizin.Patient
↑ Package               ↑ Klasse
```

Vergleich mit Python:

```python
# Python: voll qualifizierter Name
from csv import reader   # csv.reader

# Java: voll qualifizierter Name
import java.util.Scanner;  // java.util.Scanner
```

## import-Anweisungen

Mit `import` kannst du Klassen ohne Package-Präfix verwenden:

```java
import java.util.Scanner;    // Einzelne Klasse importieren
import java.util.*;          // Alle Klassen aus java.util importieren

public class MeinProgramm {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);  // statt java.util.Scanner
    }
}
```

Ohne Import müsstest du immer den voll qualifizierten Namen schreiben:

```java
java.util.Scanner sc = new java.util.Scanner(System.in);
```

## Standard-Package

Wenn du kein `package` angibst, befindet sich die Klasse im **Standard-Package** (namenlos). Das ist für kleine Testprogramme in Ordnung, für ernsthafte Projekte solltest du aber immer Packages verwenden.

## Konventionen für Package-Namen

- Domainnamen rückwärts (wie in Python): `at.htl.medizin`
- Nur Kleinbuchstaben
- Punkte als Trenner

## Zusammenfassung

### Wichtige Begriffe
- Package
- Import
- Voll qualifizierter Klassenname (FQN)

- Packages deklarieren und die Ordnerstruktur einhalten
- Klassen aus anderen Packages importieren
- Mit `javac -d` und `java -cp` arbeiten
- Ein Programm mit Packages kompilieren und ausführen
