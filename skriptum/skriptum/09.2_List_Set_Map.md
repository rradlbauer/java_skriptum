# 09.2 List, Set, Map und Implementierungen

Nachdem du die Grundlagen des Collections Frameworks kennst, lernst du jetzt die konkreten Typen und ihre wichtigsten Implementierungen kennen.

## List<T>

Eine `List<T>` ist eine **geordnete Sammlung** mit Indexzugriff – vergleichbar mit einer Python-Liste.

### ArrayList

Die **ArrayList** ist die häufigste List-Implementierung:

```java
import java.util.List;
import java.util.ArrayList;

List<String> patienten = new ArrayList<>();

// Elemente hinzufügen
patienten.add("Anna");
patienten.add("Ben");
patienten.add("Clara");

// Auf Elemente zugreifen
String erster = patienten.get(0);       // "Anna"
int anzahl = patienten.size();          // 3

// Elemente ändern
patienten.set(1, "Bernhard");           // Index 1 von "Ben" → "Bernhard"

// Elemente entfernen
patienten.remove(0);                    // Entfernt "Anna"
patienten.remove("Clara");              // Entfernt "Clara"

// Durchlaufen
for (String p : patienten) {
    System.out.println(p);
}
```

### LinkedList

Eine **LinkedList** ist eine doppelt verlinkte Liste. Sie ist schneller beim Einfügen/Entfernen am Anfang oder Ende, aber langsamer beim wahlfreien Zugriff:

```java
List<String> list = new LinkedList<>();
list.add("Erster");            // Anfügen am Ende
list.addFirst("Neuer Erster"); // Nur bei LinkedList (nicht List-Interface!)
```

**Faustregel:** Verwende `ArrayList`, außer du führst sehr viele Einfügungen/Entfernungen am Anfang der Liste durch.

## Set<T>

Ein `Set<T>` ist eine **Menge ohne Duplikate** – vergleichbar mit einem Python-Set.

### HashSet

Der **HashSet** ist die schnellste Set-Implementierung, aber die Reihenfolge ist nicht definiert:

```java
import java.util.Set;
import java.util.HashSet;

Set<String> diagnosen = new HashSet<>();
diagnosen.add("Grippe");
diagnosen.add("Grippe");    // Wird ignoriert (kein Duplikat)
diagnosen.add("Diabetes");
diagnosen.add("Fraktur");

System.out.println(diagnosen.size());  // 3 (Grippe nur einmal)

for (String d : diagnosen) {
    System.out.println(d);  // Reihenfolge nicht vorhersagbar
}
```

### TreeSet

Der **TreeSet** sortiert die Elemente (natürliche Ordnung oder via Comparator):

```java
import java.util.Set;
import java.util.TreeSet;

Set<String> sortierteDiagnosen = new TreeSet<>();
sortierteDiagnosen.add("Grippe");
sortierteDiagnosen.add("Diabetes");
sortierteDiagnosen.add("Fraktur");

for (String d : sortierteDiagnosen) {
    System.out.println(d);  // Ausgabe: Diabetes, Fraktur, Grippe (alphabetisch)
}
```

### equals() und hashCode()

Damit `HashSet` (und `HashMap`) korrekt funktionieren, müssen deine eigenen Klassen `equals()` und `hashCode()` überschreiben:

```java
public class Patient {
    private String svnr;
    private String name;

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        Patient other = (Patient) obj;
        return Objects.equals(svnr, other.svnr);
    }

    @Override
    public int hashCode() {
        return Objects.hash(svnr);
    }
}
```

Ohne korrektes `hashCode()` funktionieren HashSet und HashMap nicht zuverlässig.

## Map<K, V>

Eine `Map<K, V>` speichert **Schlüssel-Wert-Paare** – vergleichbar mit einem Python-Dict.

### HashMap

```java
import java.util.Map;
import java.util.HashMap;

Map<String, Integer> alterMap = new HashMap<>();
alterMap.put("Anna", 25);
alterMap.put("Ben", 30);
alterMap.put("Clara", 28);

// Wert abrufen
int alterAnna = alterMap.get("Anna");    // 25

// Vorhandenen Schlüssel prüfen
if (alterMap.containsKey("Ben")) {
    System.out.println("Ben ist vorhanden");
}

// Alle Einträge durchlaufen
for (Map.Entry<String, Integer> eintrag : alterMap.entrySet()) {
    System.out.println(eintrag.getKey() + ": " + eintrag.getValue() + " Jahre");
}

// Nur Schlüssel durchlaufen
for (String name : alterMap.keySet()) {
    System.out.println(name);
}

// Nur Werte durchlaufen
for (int alter : alterMap.values()) {
    System.out.println(alter);
}

// Entfernen
alterMap.remove("Clara");
```

### TreeMap

Der **TreeMap** sortiert die Einträge nach dem Schlüssel (natürliche Ordnung):

```java
Map<String, Integer> sortierteMap = new TreeMap<>();
sortierteMap.put("Zora", 22);
sortierteMap.put("Anna", 25);
sortierteMap.put("Ben", 30);

for (Map.Entry<String, Integer> e : sortierteMap.entrySet()) {
    System.out.println(e.getKey());  // Anna, Ben, Zora (alphabetisch)
}
```

## Wichtige Implementierungen im Überblick

| Interface | Implementierung | Eigenschaften |
|-----------|----------------|---------------|
| `List` | `ArrayList` | Schneller Indexzugriff, langsam beim Einfügen/Entfernen in der Mitte |
| `List` | `LinkedList` | Doppelt verlinkt, schnell beim Einfügen/Entfernen am Anfang/Ende |
| `Set` | `HashSet` | Keine Duplikate, keine Garantie für Reihenfolge |
| `Set` | `TreeSet` | Keine Duplikate, sortiert (Comparable/Comparator) |
| `Map` | `HashMap` | Schlüssel-Wert-Paare, schneller Zugriff, keine Reihenfolge |
| `Map` | `TreeMap` | Schlüssel-Wert-Paare, sortiert nach Schlüssel |

## Vergleich Python vs. Java

| Python | Java (Interface) | Java (Implementierung) |
|--------|-----------------|----------------------|
| `liste = []` | `List<T>` | `ArrayList<>`, `LinkedList<>` |
| `menge = set()` | `Set<T>` | `HashSet<>`, `TreeSet<>` |
| `dict = {}` | `Map<K, V>` | `HashMap<>`, `TreeMap<>` |
| `in liste` | `list.contains(x)` | – |
| `len(liste)` | `list.size()` | – |
| `liste[i]` | `list.get(i)` | – |

## Zusammenfassung

### Wichtige Begriffe
- `List` / `ArrayList` / `LinkedList`
- `Set` / `HashSet` / `TreeSet`
- `Map` / `HashMap` / `TreeMap`
- `equals()` / `hashCode()`
- `Map.Entry<K, V>`
- `keySet()` / `values()` / `entrySet()`

### Das sollst du können
- Die wichtigsten Implementierungen von List, Set und Map nennen und unterscheiden
- Daten in ArrayList, HashSet und HashMap speichern und abrufen
- Sets zur Entfernung von Duplikaten verwenden
- Maps für Schlüssel-Wert-Zuordnungen einsetzen
- Den Unterschied zwischen den Implementierungen erklären
